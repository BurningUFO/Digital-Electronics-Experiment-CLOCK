# CLOCK 项目方案书

## 1. 项目名称

基于 TEC-8 实验平台和 EPM7128 CPLD 的多模式智能电子钟系统设计

## 2. 项目背景

本项目来源于数字逻辑与数字系统课程设计，目标是在 TEC-8 实验平台上实现一个基于 CPLD 的电子钟系统，并在基础计时功能之上扩展实用且具有创新性的功能模块。

电子钟作为典型的数字系统实验对象，具有以下特点：

1. 结构清晰，便于进行模块化设计
2. 既包含基本计数逻辑，也包含状态控制与显示逻辑
3. 适合扩展成具有实际应用场景的系统

因此，本项目选择在基础电子钟的基础上，进一步实现多模式控制和课程作息提醒等特色功能。

## 3. 项目目标

### 3.1 基础目标

1. 实现 24 小时制电子钟计时
2. 实现时、分、秒显示
3. 实现秒、分、时正确进位
4. 实现复位功能

### 3.2 完善目标

1. 实现校时功能
2. 实现闹铃功能
3. 实现 12/24 小时制切换
4. 实现倒计时功能

### 3.3 创新目标

1. 实现课程作息提醒功能
2. 使电子钟不仅能“走时”，还能贴合学生学习场景进行提醒

## 4. 硬件平台与资源

本项目基于 TEC-8 实验平台进行实现，核心器件为：

1. `EPM7128SLC84-15`

主要使用的硬件资源包括：

1. 系统时钟输入
2. 复位输入
3. 用户按键输入
4. 六位数码管显示
5. 蜂鸣器提醒输出

显示资源说明：

1. `LG1` 采用七段译码显示
2. `LG2` 到 `LG6` 采用 `8421` 码输入显示

## 5. 系统总体方案

本项目采用模块化设计思想，将系统划分为基础计时模块、模式控制模块、显示模块和扩展功能模块。

系统总体工作模式包括：

1. 正常时钟模式
2. 校时模式
3. 闹铃模式
4. 12/24 小时显示切换模式
5. 倒计时模式
6. 课程作息提醒模式

整体上采用“公共控制框架 + 独立功能模块”的结构：

1. 公共控制框架由组长负责
2. 组员分别完成功能模块
3. 最终在顶层进行统一整合

## 6. 功能模块划分

### 6.1 基础计时模块

负责完成电子钟的基本时分秒计数以及进位逻辑，是整个系统的基础。

### 6.2 模式控制模块 `mode_ctrl`

负责不同工作模式之间的切换，并为其余模块提供模式状态。

### 6.3 按键处理模块 `key_ctrl`

负责按键消抖、边沿检测以及按键控制信号输出。

### 6.4 显示控制模块 `display_ctrl`

负责不同模式下显示内容的组织，以及闪烁和状态提示。

### 6.5 校时模块

负责对小时和分钟进行修改，实现时间校准功能。

### 6.6 闹铃模块

负责闹铃时间设置、到点比较和提醒触发。

### 6.7 12/24 小时转换模块

负责切换显示格式，使系统支持 12 小时和 24 小时两种时间显示方式。

### 6.8 倒计时模块

负责倒计时的设定、运行、结束检测与提醒输出。

### 6.9 课程作息提醒模块

负责根据预设时间表在指定时间点给出提醒，是本项目的创新扩展模块。

### 6.10 顶层集成模块 `top`

负责连接所有功能模块，形成最终可下载运行的工程。

## 7. 模块接口定义

本节用于统一各成员后续实现时的接口风格。为避免联调时反复改端口名，建议后续模块按以下接口定义设计。

### 7.1 基础时间核心模块 `time_core`

#### 功能

负责基础时分秒计数、24 小时进位和当前时间输出。

#### 建议接口

```verilog
module time_core (
    input  wire       clk_1k,
    input  wire       rst_n,
    input  wire       tick_1hz,
    input  wire       load_time_en,
    input  wire [4:0] load_hour,
    input  wire [5:0] load_min,
    input  wire [5:0] load_sec,
    output wire [4:0] cur_hour,
    output wire [5:0] cur_min,
    output wire [5:0] cur_sec,
    output wire       hour_pulse
);
```

#### 说明

1. `tick_1hz` 为 1 秒使能脉冲
2. `load_time_en` 用于校时时直接装载时间
3. `hour_pulse` 可供整点报时或提醒模块使用

### 7.2 模式控制模块 `mode_ctrl`

#### 功能

负责系统模式切换与模式状态输出。

#### 建议接口

```verilog
module mode_ctrl (
    input  wire       clk_1k,
    input  wire       rst_n,
    input  wire       key_mode_pulse,
    input  wire       key_confirm_pulse,
    output reg  [2:0] mode_state
);
```

#### 模式编码建议

```text
000 正常时钟模式
001 校时模式
010 闹铃模式
011 12/24 小时切换模式
100 倒计时模式
101 课程作息提醒模式
```

### 7.3 按键处理模块 `key_ctrl`

#### 功能

负责按键消抖、边沿检测和脉冲输出。

#### 建议接口

```verilog
module key_ctrl (
    input  wire clk_1k,
    input  wire rst_n,
    input  wire key_mode,
    input  wire key_select,
    input  wire key_add,
    input  wire key_confirm,
    output wire key_mode_pulse,
    output wire key_select_pulse,
    output wire key_add_pulse,
    output wire key_confirm_pulse
);
```

#### 建议按键定义

```text
key_mode    模式切换
key_select  选择当前设置项
key_add     当前项加一
key_confirm 确认/启停/开关
```

### 7.4 显示控制模块 `display_ctrl`

#### 功能

根据当前模式组织数码管显示内容。

#### 建议接口

```verilog
module display_ctrl (
    input  wire       clk_1k,
    input  wire       rst_n,
    input  wire [2:0] mode_state,
    input  wire [4:0] cur_hour,
    input  wire [5:0] cur_min,
    input  wire [5:0] cur_sec,
    input  wire [4:0] alarm_hour,
    input  wire [5:0] alarm_min,
    input  wire [5:0] countdown_value,
    input  wire [3:0] schedule_index,
    input  wire [5:0] schedule_hour,
    input  wire [5:0] schedule_min,
    input  wire [5:0] blink_mask,
    output wire [7:0] sec_unit_seg,
    output wire [3:0] sec_ten_bcd,
    output wire [3:0] min_unit_bcd,
    output wire [3:0] min_ten_bcd,
    output wire [3:0] hour_unit_bcd,
    output wire [3:0] hour_ten_bcd
);
```

#### 说明

1. `blink_mask` 用于控制某些位闪烁
2. 当前工程中最终输出仍需适配 `LG1` 七段译码与 `LG2-LG6` 的 `8421` 码格式

### 7.5 校时模块 `time_set_ctrl`

#### 功能

负责修改当前时间。

#### 建议接口

```verilog
module time_set_ctrl (

    output reg  [5:0] set_min,
    output reg  [5:0] set_sec,
    output reg  [5:0] blink_mask
);
```

### 7.6 闹铃模块 `alarm_ctrl`

负责闹铃时间设置、比较和提醒输出。


### 7.7 12/24 小时切换模块 `hour_format_ctrl`

#### 功能

负责当前时间的 12/24 小时显示格式切换。

#### 建议接口

```verilog
module hour_format_ctrl (
```

### 7.8 倒计时模块 `countdown_ctrl`

#### 功能

负责倒计时设置、运行和结束提醒。

#### 建议接口


### 7.9 课程作息提醒模块 `schedule_remind_ctrl`

#### 功能

根据预设课程时间点进行提醒，是本项目的创新模块。

#### 建议接口

```verilog
module schedule_remind_ctrl (
```

### 7.10 蜂鸣器控制模块 `beep_ctrl`

#### 功能

统一处理整点报时、闹铃、倒计时结束和课程作息提醒等提示请求。

#### 建议接口

```verilog
module beep_ctrl (
    input  wire clk_1k,
    input  wire rst_n,

```

## 8. 管脚分配方案

### 8.1 已确认输入引脚

| 信号名 | CPLD 管脚 | 实验箱资源 | 说明 |
| --- | --- | --- | --- |
| `rst` | `PIN_1` | `CLR#` | 低电平复位 |
| `clk_1k` | `PIN_57` | `CP2` | 当前工程按 1 kHz 使用 |

### 8.2 当前显示输出引脚

#### `LG1` 七段译码输出

| 工程信号 | CPLD 管脚 | 对应段 |
| --- | --- | --- |
| `sec_unit_seg[0]` | `PIN_44` | a |
| `sec_unit_seg[1]` | `PIN_45` | b |
| `sec_unit_seg[2]` | `PIN_46` | c |
| `sec_unit_seg[3]` | `PIN_48` | d |
| `sec_unit_seg[4]` | `PIN_49` | e |
| `sec_unit_seg[5]` | `PIN_50` | f |
| `sec_unit_seg[6]` | `PIN_51` | g |
| `sec_unit_seg[7]` | `PIN_52` | dp |

#### `LG2` 到 `LG6` 的 `8421` 码输出

实验箱当前位序按：

```text
A = bit0 = 1
B = bit1 = 2
C = bit2 = 4
D = bit3 = 8
```

| 显示位 | 工程信号 | CPLD 管脚 |
| --- | --- | --- |
| `LG2-A` | `sec_ten_bcd[0]` | `PIN_37` |
| `LG2-B` | `sec_ten_bcd[1]` | `PIN_39` |
| `LG2-C` | `sec_ten_bcd[2]` | `PIN_40` |
| `LG2-D` | `sec_ten_bcd[3]` | `PIN_41` |
| `LG3-A` | `min_unit_bcd[0]` | `PIN_35` |
| `LG3-B` | `min_unit_bcd[1]` | `PIN_36` |
| `LG3-C` | `min_unit_bcd[2]` | `PIN_17` |
| `LG3-D` | `min_unit_bcd[3]` | `PIN_18` |

| `LG4-B` | `min_ten_bcd[1]` | `PIN_31` |
| `LG4-C` | `min_ten_bcd[2]` | `PIN_33` |
| `LG4-D` | `min_ten_bcd[3]` | `PIN_34` |
| `LG5-A` | `hour_unit_bcd[0]` | `PIN_25` |
| `LG5-B` | `hour_unit_bcd[1]` | `PIN_27` |
| `LG5-C` | `hour_unit_bcd[2]` | `PIN_28` |

| `LG6-C` | `hour_ten_bcd[2]` | `PIN_22` |
| `LG6-D` | `hour_ten_bcd[3]` | `PIN_24` |

### 8.3 预留输入引脚建议

后续若需要加入额外按键，可优先从实验箱开关或按钮资源中选择 3 到 4 路输入作为：

1. 模式切换键
2. 选择键
3. 加一键
4. 确认键

具体管脚可在组长完成交互方案后统一补入 `clock.qsf`。

### 8.4 预留输出引脚建议

若后续增加蜂鸣器、LED 状态提示等功能，建议单独补充：

1. `beep_out`
2. `mode_led`
3. `alarm_led`

这些输出也由组长统一分配，避免与现有显示信号冲突。

## 10. 系统显示与交互方案

### 8.1 显示方案

1. 正常模式：显示当前 `时:分:秒`
2. 校时模式：显示当前待调整时间，被调整位闪烁
3. 闹铃模式：显示闹铃时间
4. 倒计时模式：显示当前倒计时值
5. 作息提醒模式：显示提醒时间或提醒编号

### 8.2 交互方案

建议按键功能如下：

1. 模式切换键
2. 项目选择键
3. 数值增加键
4. 确认/启动键

这样可以保证不同功能在统一交互框架下运行。

## 11. 项目创新点

本项目的主要创新点为：

### 9.1 课程作息提醒功能

在电子钟基础功能之外，引入课程作息提醒机制，根据预设时间点触发提醒，使系统更贴近学生实际使用场景。

其创新性体现在：

1. 具有明确实际应用价值
2. 区别于单一电子钟或普通闹铃
3. 体现了数字系统面向具体应用场景设计的能力

### 9.2 多模式统一控制结构

通过统一模式控制和显示控制模块，使多个功能共用同一套系统框架，提高工程结构清晰度和可扩展性。
