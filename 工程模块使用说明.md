# CLOCK 工程模块使用说明

这份文档用于说明当前主线工程中已经实现并接入的模块、输入映射、模式行为和后续模块对接方式。

## 1. 当前已接入模块

| 文件 | 模块名 | 作用 |
| --- | --- | --- |
| `clock.v` | `clock` | 顶层集成模块，负责连接所有已实现模块 |
| `clk_ring.v` | `clk_ring` | 将 `1kHz` 输入整理为 `1Hz` 计时使能脉冲 |
| `key_ctrl.v` | `key_ctrl` | 处理 `QD` 按键消抖与脉冲输出，并根据 `SWA/SWB/SWC` 译码控制动作 |
| `mode_ctrl.v` | `mode_ctrl` | 维护系统当前模式 |
| `time_set_ctrl.v` | `time_set_ctrl` | 在校时模式下切换“设置小时/设置分钟”，并生成加一脉冲 |
| `time_core.v` | `time_core` | 保存当前时间，负责正常走时和校时修改 |
| `display_ctrl.v` | `display_ctrl` | 根据当前模式决定六位显示内容 |
| `seg_7.v` | `seg_7` | 将 `LG1` 的 BCD 数字译码为七段段码 |

## 2. 当前顶层输入输出

### 输入

| 信号 | 实验箱资源 | 说明 |
| --- | --- | --- |
| `clk_1k` | `CP2 / PIN_57` | 系统时钟输入，实验时应切到 `1kHz` |
| `rst` | `CLR# / PIN_1` | 低电平复位 |
| `qd_key` | `QD / PIN_60` | 统一执行按键 |
| `sw_a` | `SWA / PIN_4` | 功能选择位 A |
| `sw_b` | `SWB / PIN_5` | 功能选择位 B |
| `sw_c` | `SWC / PIN_6` | 功能选择位 C |

### 输出

| 信号 | 实验箱资源 | 说明 |
| --- | --- | --- |
| `sec_unit_seg[7:0]` | `LG1` | 七段译码输出，显示秒个位或模式提示 |
| `sec_ten_bcd[3:0]` | `LG2` | `8421` 码 |
| `min_unit_bcd[3:0]` | `LG3` | `8421` 码 |
| `min_ten_bcd[3:0]` | `LG4` | `8421` 码 |
| `hour_unit_bcd[3:0]` | `LG5` | `8421` 码 |
| `hour_ten_bcd[3:0]` | `LG6` | `8421` 码 |

## 3. 按键与功能映射

当前工程采用“`SWA/SWB/SWC` 选功能，`QD` 执行动作”的统一输入方式。

| `SWC SWB SWA` | `ctrl_sel` | `QD` 对应动作 | 输出脉冲 |
| --- | --- | --- | --- |
| `000` | `3'b000` | 模式切换 | `key_mode_pulse` |
| `001` | `3'b001` | 选择当前设置项 | `key_select_pulse` |
| `010` | `3'b010` | 当前数值加一 | `key_add_pulse` |
| `011` | `3'b011` | 预留为确认/启停/开关 | `key_confirm_pulse` |
| `100` 到 `111` | 保留 | 暂未使用 | 无 |

## 4. 当前模式定义

| `mode_state` | 模式名 | 当前状态 |
| --- | --- | --- |
| `3'b000` | 正常时钟模式 | 已接入 |
| `3'b001` | 校时模式 | 已接入 |
| `3'b010` | 闹铃模式 | 预留 |
| `3'b011` | 12/24 小时制模式 | 预留 |
| `3'b100` | 倒计时模式 | 预留 |
| `3'b101` | 课程作息提醒模式 | 预留 |

模式切换顺序为：

`正常 -> 校时 -> 闹铃 -> 12/24 小时制 -> 倒计时 -> 作息提醒 -> 正常`

## 5. 当前可用功能说明

### 5.1 正常走时

1. 上电后按一次 `CLR#` 复位到 `00:00:00`
2. `CP2` 输入为 `1kHz` 时，系统按 24 小时制正常走时
3. `23:59:59 -> 00:00:00`

### 5.2 校时功能

1. 将 `SWA/SWB/SWC` 置为 `000`，按一次 `QD`，切到校时模式
2. 此时 `LG1` 显示当前设置字段：
   `1` 表示设置小时
   `2` 表示设置分钟
3. 将 `SWA/SWB/SWC` 置为 `001`，按一次 `QD`，在“设置小时/设置分钟”之间切换
4. 将 `SWA/SWB/SWC` 置为 `010`，按一次 `QD`，当前字段加一
5. 小时加一规则：
   `23 -> 00`
6. 分钟加一规则：
   `59 -> 00`
7. 校时模式下正常走时冻结，避免边调边走

## 6. 模块对接关系

当前主线数据流为：

`key_ctrl -> mode_ctrl -> time_set_ctrl -> time_core -> display_ctrl -> seg_7`

各模块职责边界如下：

1. `key_ctrl` 只输出控制脉冲，不直接改时间
2. `mode_ctrl` 只维护模式状态，不直接改显示
3. `time_set_ctrl` 只决定“当前调哪个字段”和“发出哪个加一脉冲”
4. `time_core` 是唯一保存当前时间的模块
5. `display_ctrl` 是唯一决定“当前显示什么”的模块

## 7. 后续模块接入建议

### 闹铃模块 `alarm_ctrl`

建议读取：

1. `mode_state` 或 `mode_alarm`
2. `key_select_pulse`
3. `key_add_pulse`
4. `key_confirm_pulse`
5. `time_core` 输出的当前时间

### 12/24 小时模块 `hour_format_ctrl`

建议不要直接改 `time_core` 内部时间，只修改送入 `display_ctrl` 的小时显示值。

### 倒计时模块 `countdown_ctrl`

建议维护独立倒计时寄存器，并通过 `display_ctrl` 在倒计时模式下接管显示。

### 作息提醒模块 `schedule_remind_ctrl`

建议读取当前时间并输出提醒事件，不直接改底层计时。

## 8. 当前工程注意事项

1. `LG1` 为七段译码，`LG2-LG6` 为 `8421` 码，不能混用
2. 实验箱当前位序按 `A=1, B=2, C=4, D=8` 连接
3. `sec_unit_seg[7]` 对应小数点 `dp`，当前未使用
4. `sec_ten_bcd[3]` 常为 `0` 是正常现象，因为秒十位最大只到 `5`
5. 后续新增模块时，应优先新增独立 `.v` 文件，再通过 `clock.v` 顶层接入
