# CLOCK 工程模块使用说明

这份文档用于说明当前主线工程中已经实现并接入的模块、输入映射、模式行为和后续模块对接方式。

## 1. 当前已接入模块

| 文件 | 模块名 | 作用 |
| --- | --- | --- |
| `clock.v` | `clock` | 顶层集成模块，负责连接所有已实现模块 |
| `clk_ring.v` | `clk_ring` | 将 `1kHz` 输入整理为 `1Hz` 计时使能脉冲 |
| `key_ctrl.v` | `key_ctrl` | 处理 `QD` 按键消抖与脉冲输出，并根据 `SWA/SWB/SWC` 译码控制动作 |
| `mode_ctrl.v` | `mode_ctrl` | 根据独立模式开关的优先级实时生成 `mode_state` |
| `time_set_ctrl.v` | `time_set_ctrl` | 在校时模式下切换“设置小时/设置分钟”，并生成加一脉冲 |
| `time_core.v` | `time_core` | 保存当前时间，负责正常走时和校时修改 |
| `alarm_ctrl.v` | `alarm_ctrl` | 维护闹铃时间、闹铃开关状态，并完成当前时分实时比较与提醒输出 |
| `countdown_ctrl.v` | `countdown_ctrl` | 维护独立倒计时寄存器，负责倒计时设置、启动、暂停和到零结束 |
| `display_ctrl.v` | `display_ctrl` | 根据当前模式决定六位显示内容 |
| `seg_7.v` | `seg_7` | 将 `LG1` 的 BCD 数字译码为七段段码 |

## 3. 模式与按键映射

### 3.1 模式选择开关规则

当前工程采用“独立模式开关直接选模式”的方式，不再使用 `QD` 轮流切模式。

| 优先级 | 模式开关 | `mode_state` | 对应模式 |
| --- | --- | --- | --- |
| 最高 | `k5_mode_schedule = 1` | `3'b101` | 作息提醒模式 |
| 2 | `k4_mode_countdown = 1` | `3'b100` | 倒计时模式 |
| 3 | `k3_mode_hour_format = 1` | `3'b011` | 12/24 小时制模式 |
| 4 | `k2_mode_alarm = 1` | `3'b010` | 闹铃模式 |
| 5 | `k1_mode_time_set = 1` | `3'b001` | 校时模式 |
| 默认 | `K1~K5` 全为 `0` | `3'b000` | 正常时钟模式 |

举例：

1. 只打开 `K4`，系统进入倒计时模式。
2. 同时打开 `K1` 和 `K4`，由于 `K4` 优先级更高，系统仍进入倒计时模式。
3. 同时打开 `K5` 和其他任意模式开关，系统都进入作息提醒模式。

### 3.2 `QD` 功能映射

当前工程采用“`SWA/SWB/SWC` 选动作，`QD` 执行动作”的统一输入方式。

| `SWC SWB SWA` | `ctrl_sel` | `QD` 对应动作 | 输出脉冲 |
| --- | --- | --- | --- |
| `000` | `3'b000` | 当前未分配公共动作 | 无 |
| `001` | `3'b001` | 选择当前设置项 | `key_select_pulse` |
| `010` | `3'b010` | 当前数值加一 | `key_add_pulse` |
| `011` | `3'b011` | 确认 / 启停 / 开关 | `key_confirm_pulse` |
| `100` 到 `111` | 保留给特定模块 | 无公共脉冲 | 无 |

说明：

1. `key_ctrl` 现在不再输出 `key_mode_pulse`，因为模式切换已经由独立开关负责。
2. 倒计时模块为了节省资源，仍会直接读取 `ctrl_sel`，并在倒计时模式下额外使用 `100`、`101` 这两个编码。
3. 代码中的编码顺序是 `ctrl_sel = {sw_c, sw_b, sw_a}`，也就是按 `SWC SWB SWA` 理解，而不是 `SWA SWB SWC`。

## 4. 当前资源与重构状态

当前主线已经把模式切换交互从“内部状态机循环切换”重构为“独立开关优先级选模式”，并完成了闹铃设置、实时到点比较和提醒输出。本地 Quartus 全编译结果为：

1. `0 errors`
2. Fitter 成功
3. Timing 通过
4. 资源占用 `124 / 128` macrocells
5. 当前 warning 主要为 Quartus Classic Timing Analyzer 对 MAX7000S 器件家族的常规提示

本轮与当前主线资源压缩和闹铃接入相关的关键调整如下：

1. `mode_ctrl.v`
   - 原先做法：由 `key_mode_pulse` 驱动寄存器状态机，按一下 `QD` 才切换一次模式
   - 当前做法：改为对 `K1~K5` 做组合优先级解码，实时输出 `mode_state`
   - 作用：模式开关位置就是当前模式，不需要记忆“刚才切到了哪一态”
2. `key_ctrl.v`
   - 原先做法：输出 `key_mode_pulse / key_select_pulse / key_add_pulse / key_confirm_pulse`
   - 当前做法：删除 `key_mode_pulse`，只保留与当前模式内动作有关的控制脉冲
   - 作用：让 `QD` 只负责执行当前模式内的操作，不再兼任模式轮转
3. `clock.v`
   - 原先做法：`mode_ctrl` 读取 `key_mode_pulse`
   - 当前做法：`mode_ctrl` 读取五路独立模式开关输入
   - 作用：顶层交互职责更清晰，模式入口和功能执行分离
4. `clock.qsf`
   - 新增 `K1~K5` 对应的模式选择输入管脚约束
   - 作用：把模式选择从原先的三位动作开关中独立出来，便于上板直接操作
5. `alarm_ctrl.v`
   - 原型来源：方睿锦分支中的闹铃设置原型
   - 当前做法：按主线接口重构为独立模块，并把闹铃设置方式改成“直接字段映射”以节省宏单元
   - 作用：在不破坏当前主线结构的前提下接入闹铃基础版，并把资源占用控制在 `128` 个宏单元以内
6. `key_ctrl.v`
   - 当前做法：进一步把 `QD` 消抖稳定计数压缩为 `1` 位门限计数
   - 作用：在不改变统一按键接口形式的前提下继续释放公共控制模块资源
7. `seg_7.v`
   - 当前做法：对非法 BCD 输入 `10~15` 使用 `don't care`
   - 作用：让综合器进一步压缩七段译码逻辑，帮助闹铃基础版在当前器件上拟合通过
8. `time_core.v`
   - 优化前写法：把“秒进位、分进位、时进位、校时小时加一、校时分钟加一”全部压在一个深层嵌套的 `always` 中，依赖多层串联 `if-else` 完成进位传播
   - 优化后写法：拆成“秒计数块、分计数块、时计数块”三个独立时序块，并用 `tick_en / sec_wrap / min_wrap / hour_en` 这些共享使能信号传递进位关系
   - 作用：把原先多级串联判断改成共享条件驱动的分段计数，明显降低 `time_core` 的组合层级和 pterm 压力
9. `countdown_ctrl.v`
   - 优化前写法：在一个大 `always` 中同时处理四位数码的设置、自减、借位和到零停止，递减路径是四层嵌套借位判断
   - 优化后写法：保留 `countdown_run` 独立控制，再把 `MM:SS` 四位拆成四个独立时序块，使用 `edit_min_ten / edit_min_unit / edit_sec_ten / edit_sec_unit` 和 `borrow_sec_ten / borrow_min_unit / borrow_min_ten` 共享借位链
   - 作用：把“深层递减嵌套”改成“共享借位链 + 各位独立更新”，综合后显著降低倒计时模块宏单元占用
10. `alarm_ctrl.v` + `clock.v`
   - 当前做法：在 `alarm_ctrl.v` 中新增 `cur_alarm_bus / set_alarm_bus` 打包比较总线，并使用 `~|(cur_alarm_bus ^ set_alarm_bus)` 完成当前时分与闹铃时分的统一比较；在 `clock.v` 中把 `alarm_match` 直接接到 `sec_unit_seg[7]`
   - 作用：在不新增额外提醒端口和复杂保持状态机的前提下，把“闹铃实时到点比较 + 提醒输出”补回主线
11. 本轮功能补回结果
   - 量化结果：主线在前一版 `118 / 128` 的基础上补回闹铃比较与提醒后，最终资源占用为 `124 / 128`

## 5. 当前可用功能说明

### 5.1 正常走时

1. 保持 `K1~K5` 全部为 `0`
2. 上电后按一次 `CLR#` 复位到 `00:00:00`
3. `CP2` 输入为 `1kHz` 时，系统按 24 小时制正常走时
4. `23:59:59 -> 00:00:00`

### 5.2 校时功能

1. 打开 `K1`，进入校时模式
2. 此时 `LG1` 显示当前设置字段：
   - `1` 表示设置小时
   - `2` 表示设置分钟
3. 将 `SWA/SWB/SWC` 拨成 `001`，按一次 `QD`，在“设置小时/设置分钟”之间切换
4. 将 `SWA/SWB/SWC` 拨成 `010`，按一次 `QD`，当前字段加一
5. 小时加一规则：`23 -> 00`
6. 分钟加一规则：`59 -> 00`
7. 退出校时模式后，`select_hour` 会自动回到默认的“设置小时”

### 5.3 倒计时功能

1. 打开 `K4`，进入倒计时模式
2. 倒计时模式下显示固定为 `00:MM:SS`
   - `LG6`、`LG5` 固定显示 `0`
   - `LG4~LG1` 显示倒计时时间 `MM:SS`
3. 为了控制资源占用，倒计时模式没有再额外保存“当前选中字段”状态，而是直接用 `SWC SWB SWA` 指定要修改的位
4. 倒计时设置时，按键映射如下：
   - `001 + QD`：分钟十位加一
   - `010 + QD`：分钟个位加一
   - `100 + QD`：秒十位加一
   - `101 + QD`：秒个位加一
5. 秒十位加一范围限制为 `0~5`
6. `011 + QD`：若当前倒计时不为 `00:00`，则开始倒计时
7. 倒计时运行中再次执行 `011 + QD` 会暂停倒计时
8. 倒计时减到 `00:00` 时自动停止

### 5.4 闹铃功能

1. 打开 `K2`，进入闹铃模式
2. 闹铃模式下显示为 `HH:MM:E0`
   - `LG6`、`LG5` 显示闹铃小时
   - `LG4`、`LG3` 显示闹铃分钟
   - `LG2` 显示闹铃开关位 `E`
   - `LG1` 固定显示 `0`
3. 为节省宏单元，当前闹铃模式不再额外维护“当前字段选择态”，而是直接按功能编码修改目标字段
4. 按键映射如下：
   - `001 + QD`：闹铃小时加一
   - `010 + QD`：闹铃分钟加一
   - `011 + QD`：闹铃开关翻转
5. 小时加一规则：`23 -> 00`
6. 分钟加一规则：`59 -> 00`
7. `LG2` 当前显示闹铃开关位：
   - `0` 表示闹铃关闭
   - `1` 表示闹铃开启
8. 当前闹铃模式的实际显示格式更准确地说是 `HH:MM:E0`
   - `E` 为闹铃使能位，位于 `LG2`
   - `LG1` 固定显示 `0`
9. 当闹铃开关为 `1` 且当前时间的“小时 + 分钟”等于设定闹铃时间时，`LG1.dp` 会点亮作为提醒输出
10. 当前提醒输出不依赖是否正停留在闹铃模式，只要闹铃开启且当前时分匹配，`dp` 都会亮
11. 为控制宏单元占用，当前主线采用“当前时分持续比较”的方案，没有额外加入秒级锁存、蜂鸣保持和消音状态机
12. 因此当前提醒效果表现为：在匹配的整个分钟内，`LG1` 数字本体仍显示原来的 `0`，只有小数点 `dp` 持续点亮

## 6. 模块对接关系

当前主线数据流为：

`K1~K5 -> mode_ctrl`

`QD + SWA/SWB/SWC -> key_ctrl`

`mode_ctrl + key_ctrl -> (time_set_ctrl / alarm_ctrl / countdown_ctrl) -> (time_core / alarm state / countdown state) -> display_ctrl -> seg_7`

各模块职责边界如下：

1. `mode_ctrl` 只负责根据模式开关优先级生成 `mode_state`
2. `key_ctrl` 只负责输出当前模式内的控制脉冲和 `ctrl_sel`
3. `time_set_ctrl` 只决定“当前调哪个字段”和“发出哪个加一脉冲”
4. `time_core` 是唯一保存当前时间的模块
5. `alarm_ctrl` 是唯一保存闹铃状态和闹铃匹配结果的模块
6. `countdown_ctrl` 是唯一保存倒计时状态的模块
7. `display_ctrl` 是唯一决定“当前显示什么”的模块

## 7. 后续模块接入建议

### 闹铃模块 `alarm_ctrl`

当前主线已经接入基础版，当前职责为：

1. 保存闹铃小时和分钟
2. 保存闹铃开关状态
3. 在闹铃模式下响应 `001/010/011 + QD`
4. 对当前时分与设定闹铃时分做实时比较
5. 通过 `display_ctrl` 显示闹铃时间，并通过 `LG1.dp` 输出提醒信号

后续若继续扩展，建议优先补：

1. 消音或提醒保持策略
2. 蜂鸣器或独立 LED 等更强提醒输出
3. 秒级触发策略或多组闹铃扩展

### 12/24 小时模块 `hour_format_ctrl`

建议不要直接改 `time_core` 内部时间，只修改送入 `display_ctrl` 的小时显示值。

### 倒计时模块 `countdown_ctrl`

当前主线已经按独立模块方式接入：

1. 读取 `mode_countdown`
2. 读取 `qd_pulse` 和 `ctrl_sel`
3. 使用 `tick_1h` 作为倒计时递减使能
4. 内部维护 `MM:SS` 四位 BCD 寄存器
5. 为节省宏单元，设置阶段直接按 `ctrl_sel` 选择要修改的位，而不是额外维护字段选择寄存器输出
6. 通过 `display_ctrl` 在倒计时模式下接管显示

### 作息提醒模块 `schedule_remind_ctrl`

建议读取当前时间并输出提醒事件，不直接改底层计时。

## 8. 当前工程注意事项

1. `LG1` 为七段译码，`LG2-LG6` 为 `8421` 码，不能混用
2. 实验箱当前位序按 `A=1, B=2, C=4, D=8` 连接
3. `sec_unit_seg[7]` 对应小数点 `dp`，当前已用作闹铃提醒输出
4. `sec_ten_bcd[3]` 常为 `0` 是正常现象，因为秒十位最大只到 `5`
5. 倒计时模式下 `LG6`、`LG5` 固定为 `0`，显示格式为 `00:MM:SS`
6. 闹铃模式下当前显示格式为 `HH:MM:E0`
7. 当前模式由 `K1~K5` 决定，不再通过 `QD` 轮流切换
8. 若多个模式开关同时打开，以高位优先级为准
9. 闹铃开启且当前时分匹配时，`LG1.dp` 会持续点亮一个匹配分钟
10. 后续新增模块时，应优先新增独立 `.v` 文件，再通过 `clock.v` 顶层接入
