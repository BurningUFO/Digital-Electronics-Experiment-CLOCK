# CLOCK 工程模块使用说明

这份文档用于说明当前主线工程中已经实现并接入的模块、输入映射、模式行为和后续模块对接方式。

## 1. 当前已接入模块

| 文件 | 模块名 | 作用 |
| --- | --- | --- |
| `clock.v` | `clock` | 顶层集成模块，负责连接所有已实现模块 |
| `clk_ring.v` | `clk_ring` | 将 `1kHz` 输入整理为 `1Hz` 计时使能脉冲 |
| `key_ctrl.v` | `key_ctrl` | 处理 `QD` 按键消抖与脉冲输出，并根据 `SWA/SWB/SWC` 译码控制动作 |
| `mode_ctrl.v` | `mode_ctrl` | 维护系统当前模式 |
| `time_set_ctrl.v` | `time_set_ctrl` | 在校时模式下切换“设置小时/设置分钟”，并生成加一脉冲 |
| `time_core.v` | `time_core` | 保存当前时间，负责正常走时和校时修改 |
| `countdown_ctrl.v` | `countdown_ctrl` | 维护独立倒计时寄存器，负责倒计时设置、启动、暂停和到零结束 |
| `display_ctrl.v` | `display_ctrl` | 根据当前模式决定六位显示内容 |
| `seg_7.v` | `seg_7` | 将 `LG1` 的 BCD 数字译码为七段段码 |

## 3. 按键与功能映射

当前工程采用“`SWA/SWB/SWC` 选功能，`QD` 执行动作”的统一输入方式。

| `SWC SWB SWA` | `ctrl_sel` | `QD` 对应动作 | 输出脉冲 |
| --- | --- | --- | --- |
| `000` | `3'b000` | 模式切换 | `key_mode_pulse` |
| `001` | `3'b001` | 选择当前设置项 | `key_select_pulse` |
| `010` | `3'b010` | 当前数值加一 | `key_add_pulse` |
| `011` | `3'b011` | 预留为确认/启停/开关 | `key_confirm_pulse` |
| `100` 到 `111` | 保留 | 暂未使用 | 无 |

说明：基础公共框架里 `100~111` 仍不输出标准控制脉冲；当前倒计时模块为了节省资源，会直接读取 `ctrl_sel`，并在倒计时模式下额外使用 `100`、`101` 这两个编码。

## 3.1 当前资源优化状态

当前主线已经针对 `EPM7128SLC84-15` 的 `128` 个宏单元做过一轮面积压缩优化，本地 Quartus 全编译结果为：

1. `0 errors`
2. Fitter 成功
3. Timing 通过
4. 资源占用 `121 / 128` macrocells

本轮资源优化的文档贡献人统一按小组全体记录：

1. 杨龙驹
2. 蒲佳鑫
3. 樊逸晨
4. 方睿锦

本轮释放宏单元的核心做法如下：

1. `clock.qsf`
   - 将综合方向改为 `OPTIMIZATION_TECHNIQUE AREA`
   - 打开 `AUTO_RESOURCE_SHARING ON`
   - 作用：让 Quartus 在综合和拟合阶段优先考虑面积而不是速度，并尽量共享重复逻辑
2. `mode_ctrl.v`
   - 原先做法：除 `mode_state` 外，还长期导出 `mode_normal`、`mode_time_set`、`mode_alarm`、`mode_hour_format`、`mode_countdown`、`mode_schedule` 等多路模式比较结果
   - 当前做法：只保留 `mode_state`
   - 作用：把“一个状态寄存器 + 多路并行比较输出”收缩成“一个状态寄存器输出”，减少重复比较逻辑和高扇出信号
3. `clock.v`
   - 原先做法：直接接收 `mode_ctrl` 导出的整组模式使能信号
   - 当前做法：仅对当前真正需要使用的两路模式做顶层按需解码：
     - `mode_time_set = (mode_state == 3'b001)`
     - `mode_countdown = (mode_state == 3'b100)`
   - 作用：只保留现阶段必需的模式比较，避免未使用模式信号长期占用资源
4. `display_ctrl.v`
   - 原先做法：使用 `always @(*) + reg` 写多路显示选择
   - 当前做法：改为连续赋值形式的组合选择
   - 作用：减少中间选择网络和不必要的组合描述开销，让综合结果更紧凑
5. `key_ctrl.v`
   - 原先做法：消抖计数器使用 `4` 位，稳定门限为 `15`
   - 当前做法：消抖计数器压缩为 `3` 位，稳定门限改为 `7`
   - 作用：以更小的计数器和比较器实现基础消抖功能，降低公共控制模块的宏单元占用

## 4. 当前模式定义

| `mode_state` | 模式名 | 当前状态 |
| --- | --- | --- |
| `3'b000` | 正常时钟模式 | 已接入 |
| `3'b001` | 校时模式 | 已接入 |
| `3'b010` | 闹铃模式 | 预留 |
| `3'b011` | 12/24 小时制模式 | 预留 |
| `3'b100` | 倒计时模式 | 已接入 |
| `3'b101` | 课程作息提醒模式 | 预留 |

模式切换顺序为：

`正常 -> 校时 -> 闹铃 -> 12/24 小时制 -> 倒计时 -> 作息提醒 -> 正常`

## 5. 当前可用功能说明

### 5.1 正常走时

1. 上电后按一次 `CLR#` 复位到 `00:00:00`
2. `CP2` 输入为 `1kHz` 时，系统按 24 小时制正常走时
3. `23:59:59 -> 00:00:00`

### 5.2 校时功能

1. 将 `SWA/SWB/SWC` 置为 `000`，按一次 `QD`，切到校时模式
2. 此时 `LG1` 显示当前设置字段：
   `1` 表示设置小时
   `2` 表示设置分钟
3. 将 `SWA/SWB/SWC` 置为 `001`，按一次 `QD`，在“设置小时/设置分钟”之间切换
4. 将 `SWA/SWB/SWC` 置为 `010`，按一次 `QD`，当前字段加一
5. 小时加一规则：
   `23 -> 00`
6. 分钟加一规则：
   `59 -> 00`
7. 校时模式下正常走时冻结，避免边调边走

### 5.3 倒计时功能

1. 连续按模式切换直到 `mode_state = 3'b100`，进入倒计时模式
2. 倒计时模式下显示固定为 `00:MM:SS`
   - `LG6`、`LG5` 固定显示 `0`
   - `LG4~LG1` 显示倒计时时间 `MM:SS`
3. 为了控制资源占用，倒计时模式没有再额外保存“当前选中字段”状态，而是直接用 `SWC SWB SWA` 指定要修改的位
4. 倒计时设置时，按键映射如下：
   - `001 + QD`：分钟十位加一
   - `010 + QD`：分钟个位加一
   - `100 + QD`：秒十位加一
   - `101 + QD`：秒个位加一
5. 秒十位加一范围限制为 `0~5`，修正了原型分支中可设置到非法值的问题
6. `011 + QD`：若当前倒计时不为 `00:00`，则开始倒计时
7. 倒计时运行中再次执行 `011 + QD` 会暂停倒计时
8. 倒计时减到 `00:00` 时自动停止

## 6. 模块对接关系

当前主线数据流为：

`key_ctrl -> mode_ctrl -> (time_set_ctrl / countdown_ctrl) -> (time_core / countdown state) -> display_ctrl -> seg_7`

各模块职责边界如下：

1. `key_ctrl` 统一输出控制脉冲；倒计时模块额外复用了它导出的 `qd_pulse` 和 `ctrl_sel`
2. `mode_ctrl` 只维护模式状态，不直接改显示
3. `time_set_ctrl` 只决定“当前调哪个字段”和“发出哪个加一脉冲”
4. `time_core` 是唯一保存当前时间的模块
5. `countdown_ctrl` 是唯一保存倒计时状态的模块
6. `display_ctrl` 是唯一决定“当前显示什么”的模块
7. 为节省宏单元，模式的局部使能信号优先在需要它们的上层模块内按需解码，而不是由 `mode_ctrl` 长期整组导出

## 7. 后续模块接入建议

### 闹铃模块 `alarm_ctrl`

建议读取：

1. `mode_state` 或 `mode_alarm`
2. `key_select_pulse`
3. `key_add_pulse`
4. `key_confirm_pulse`
5. `time_core` 输出的当前时间

### 12/24 小时模块 `hour_format_ctrl`

建议不要直接改 `time_core` 内部时间，只修改送入 `display_ctrl` 的小时显示值。

### 倒计时模块 `countdown_ctrl`

当前主线已经按独立模块方式接入：

1. 读取 `mode_countdown`
2. 读取 `qd_pulse` 和 `ctrl_sel`
3. 使用 `tick_1h` 作为倒计时递减使能
4. 内部维护 `MM:SS` 四位 BCD 寄存器
5. 为节省宏单元，设置阶段直接按 `ctrl_sel` 选择要修改的位，而不是额外维护字段选择寄存器输出
6. 通过 `display_ctrl` 在倒计时模式下接管显示

### 作息提醒模块 `schedule_remind_ctrl`

建议读取当前时间并输出提醒事件，不直接改底层计时。

## 8. 当前工程注意事项

1. `LG1` 为七段译码，`LG2-LG6` 为 `8421` 码，不能混用
2. 实验箱当前位序按 `A=1, B=2, C=4, D=8` 连接
3. `sec_unit_seg[7]` 对应小数点 `dp`，当前未使用
4. `sec_ten_bcd[3]` 常为 `0` 是正常现象，因为秒十位最大只到 `5`
5. 倒计时模式下 `LG6`、`LG5` 固定为 `0`，显示格式为 `00:MM:SS`
6. 倒计时设置使用 `001/010/100/101 + QD` 直接修改四个字段，不再通过“字段选择态”间接修改
7. 后续新增模块时，应优先新增独立 `.v` 文件，再通过 `clock.v` 顶层接入
8. 后续新增功能若再次逼近资源上限，应优先检查是否引入了额外状态寄存器、重复模式比较逻辑和大范围多路选择网络
