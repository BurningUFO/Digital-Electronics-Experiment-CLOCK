# CLOCK 优化重构协作开发工作日志

本文档用于记录 CLOCK 项目后续使用多个 agent 协作开发时的工作说明、分工边界、协作次序和修改日志。

本日志只面向当前 `clock_amd` 主工作区：

- Vivado 工程入口：`clock_amd.xpr`
- 主线 HDL 源码目录：`clock_amd.srcs/sources_1/new/`
- 主线约束目录：`clock_amd.srcs/constrs_1/new/`
- 文档目录：`docs/`

## 1. 日志记录规则

1. 本日志不按日期作为修改条目标题。
2. 每次修改按协作顺序记录为“第 N 次修改”。
3. 若协作次序为杨龙驹、蒲佳鑫、樊逸晨，则日志依次记录为：
   - 第一次修改：杨龙驹做了什么
   - 第二次修改：蒲佳鑫做了什么
   - 第三次修改：樊逸晨做了什么
4. 每次修改都要写清楚修改人、负责 agent、修改文件、修改内容、测试情况、影响范围和对接对象。
5. 普通功能新增、接线、修复类日志按简洁条目记录。
6. 资源优化、失败尝试、接口重构、跨模块联调等内容，需要补充“原因 / 取舍 / 影响范围”。
7. 若某次修改会影响其他组员或其他 agent 的模块，必须在“对接说明”中写清楚。
8. 若修改内容与 `docs/优化重构项目书.md` 中已确认方案不一致，必须先更新方案或在日志中说明原因。

## 2. Agent 协作角色

后续计划使用一个组长 agent 和三个组员 agent 协作开发。

| Agent 角色 | 对应成员 | 职责定位 |

| 组长 agent | 杨龙驹（组长） | 负责总体架构、主线集成、接口裁决、显示与交互底座、特色功能整合、最终联调 |

## 4. 协作边界

1. 组员 agent 优先在自己负责模块内修改。
2. 需要改动 `clock.v`、`clock_amd_top.v`、`.xdc`、OLED 主 UI、统一显示出口或统一提醒仲裁时，默认由组长 agent 负责，或由组长 agent 最终整合。
3. 组员模块需要新增接口时，应先在日志中写清楚接口含义，再由组长 agent 接入主线。
4. 不直接修改 Vivado 自动生成目录。
5. 只关注主线源码、主线约束和文档。
6. 代码行为与文档不一致时，以当前代码行为为准，并在日志中记录需要同步的文档项。
