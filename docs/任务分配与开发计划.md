# CLOCK 项目任务分配与开发计划

## 1. 项目名称

基于 TEC-8 实验平台与 EPM7128 CPLD 的多模式智能电子钟系统

## 2. 小组成员

1. 杨龙驹
2. 蒲佳鑫
3. 樊逸晨
4. 方睿锦

## 3. 分工原则

本项目采用“组长负责总体架构与核心公共模块，组员负责功能模块实现，最终统一集成”的方式开展。

分工目标如下：

1. 每位成员都有明确且独立的功能模块
2. 各模块之间接口清晰，便于并行开发
3. 组长负责系统框架、整合和特色功能，保证最终版本稳定
4. 每位成员都应提交代码、说明和测试结果，便于贡献度评估

## 5. 各成员任务说明

### 5.1 杨龙驹（组长）

负责内容：

1. 顶层集成模块 `top`
2. 模式控制模块 `mode_ctrl`
3. 按键处理模块 `key_ctrl`
4. 显示控制模块 `display_ctrl`
5. 课程作息提醒模块 `schedule_remind_ctrl`
6. 最终版本整合与验收准备

重点贡献：

1. 搭建整个系统框架
2. 统一控制逻辑和显示逻辑
3. 设计项目创新功能
4. 负责所有成员模块的最终整合

### 5.2 蒲佳鑫

负责内容：

1. 12 小时制与 24 小时制切换模块 `hour_format_ctrl`
2. 倒计时模块 `countdown_ctrl`

建议目标：

1. 支持在正常模式下切换 12/24 小时显示方式
2. 倒计时结束时能触发提示信号
3. 与显示控制模块统一对接，不直接修改顶层显示输出

### 5.3 樊逸晨

负责内容：

1. 校时模块 `time_set_ctrl`
2. 时间修改过程中的状态控制

建议目标：

1. 能够调整小时和分钟
2. 支持选择被修改项
3. 校时过程中与显示闪烁逻辑配合

### 5.4 方睿锦

负责内容：

1. 闹铃模块 `alarm_ctrl`
2. 闹铃时间设置与触发

建议目标：

1. 支持设置一组闹铃时间
2. 到点后触发提醒信号
3. 支持闹铃功能开关

## 6. 模块接口协作要求

为避免多人同时修改同一部分代码，要求如下：

1. 组员原则上不要直接改 `top` 顶层框架，由组长统一整合
2. 每位成员实现自己的模块时，应先明确输入输出接口
3. 新增功能模块优先通过接口接入，不要直接覆盖已有逻辑
4. 若某个功能会影响显示或模式切换，需提前告知组长

## 7. 开发顺序建议

建议按如下顺序推进：

1. 组长先完成 `key_ctrl`、`mode_ctrl`、`display_ctrl` 基础框架
2. 樊逸晨完成校时模块并接入模式控制
3. 方睿锦完成闹铃模块并接入提醒输出
4. 蒲佳鑫完成 12/24 小时转换和倒计时模块
5. 组长实现课程作息提醒模块
6. 全组进入联调、测试与文档完善阶段

## 8. 阶段性开发计划

### 第一阶段：基础框架确定

目标：

1. 固定顶层结构
2. 确定按键定义
3. 确定显示模式切换逻辑

### 第二阶段：基础功能补齐

目标：

1. 完成校时功能
2. 完成闹铃功能
3. 完成 12/24 小时转换

### 第三阶段：扩展功能实现

目标：

1. 完成倒计时功能
2. 完成课程作息提醒模块

### 第四阶段：联调与验收准备

目标：

1. 完成各模块集成
2. 完成实验箱联调
3. 完成功能日志、测试记录与报告材料整理

## 9. 协作规范

所有成员必须遵守：

1. 每次开发前先查看 `组员Git协作指南.md`
2. 每次修改后及时更新 `功能与修改日志.md`
3. 不直接在 `main` 分支开发
4. 提交前先自行检查功能是否正常
5. 影响他人模块时要先沟通

## 10. 组长职责补充说明

作为组长，杨龙驹除技术开发外，还负责：

1. 维护主分支
2. 统一接口标准
3. 审核组员功能接入方式
4. 组织联调
5. 汇总日志、方案书和答辩材料

因此组长贡献不仅体现在代码量，也体现在总体方案设计、系统整合和项目管理能力上。
