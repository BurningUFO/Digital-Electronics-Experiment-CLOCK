## 1. 这块模块是什么

这不是裸屏，而是已经带了驱动和外围电路的 OLED 模块。

你真正面对的是：

- 一个 `SSD1306` 控制器
- 一个 `128x64` 点阵 OLED 面板
- 一个做成 `4 针 IIC` 接口的模块板

所以你使用时不需要去直连裸屏 30Pin。

## 2. 引脚定义

四个引脚分别是：

- `GND`
  - 电源地
- `VCC`
  - 模块供电
- `SCL`
  - I2C 时钟
- `SDA`
  - I2C 数据

这块板已经是 IIC 版本，不需要你额外接：

- `RES`
- `DC`
- `CS`

它们已经在模块板内部固定到了适合 IIC 模式的连接方式上。

## 3. 最重要的电气结论

这块模块可以直接接 `Nexys A7-100T`，但必须遵守两条规则：

- `VCC` 接 `3.3V`
- `SCL/SDA` 必须按 I2C 开漏方式使用

### 3.1 为什么不能默认接 5V

模块资料写的是：

- `VCC = 3V ~ 5.5V`

这句话的意思不是“你在 FPGA 上可以随便 5V 直连”。

因为模块板上的 `SCL/SDA` 会被上拉到模块的 `VCC`。

如果你：

- 给模块供 `5V`
- 又把 `SCL/SDA` 直接接 FPGA

那总线高电平也会变成 `5V`，这对 `Nexys A7` 的 FPGA IO 不安全。

所以对 `Nexys A7` 唯一推荐的接法是：

- `VCC -> 3.3V`

### 3.2 为什么 I2C 不能当普通 GPIO

I2C 总线不是普通推挽输出线。

标准 I2C 的行为是：

- 需要发 `0` 时，主动把线拉低
- 需要发 `1` 时，不是主动输出高，而是释放总线
- 让上拉电阻把线拉高

所以在 FPGA 里：

- `SCL`
- `SDA`

最适合做成：

- `inout`
- 输出 `0`
- 或输出高阻 `Z`

这就是“开漏”思路。

## 4. 物理接线

当前最小测试工程使用的接线是：

- `OLED GND -> Nexys GND`
- `OLED VCC -> Nexys 3.3V`
- `OLED SCL -> JB1`
- `OLED SDA -> JB2`

对应 FPGA 引脚：

- `JB1 -> D14`
- `JB2 -> F16`

推荐用 4 根杜邦线分开接，不要因为 Pmod 排针顺手就整排插上去。

## 5. 这块 OLED 的颜色为什么是“上黄下蓝”

你现在看到的是黄蓝双色版本。

它的物理特性通常是：

- 上方约 `1/4` 区域发黄光
- 下方约 `3/4` 区域发蓝光

这不是程序控制出来的，也不是初始化寄存器决定的，而是面板本身的发光材料分区。

所以：

- 你可以控制像素亮不亮
- 但不能用程序把黄区改成蓝区，或把蓝区改成白区

## 6. I2C 地址怎么理解

资料里常见两种写法：

- `0x78`
- `0x7A`

这其实是 `8 位写地址` 的写法。

而很多软件库用的是 `7 位地址` 写法：

- `0x3C`
- `0x3D`

它们之间的关系是：

- `0x3C << 1 = 0x78`
- `0x3D << 1 = 0x7A`

对于你这块模块，默认大概率是：

- `7-bit 地址 = 0x3C`
- 写地址字节 = `0x78`

如果接线都对但完全没响应，再考虑是否切到了另一档地址。

## 7. SSD1306 最关键的显示知识

### 7.1 屏幕分辨率

这块模块是：

- `128` 列
- `64` 行

### 7.2 显存组织

`SSD1306` 的内部显示 RAM 常按“页”组织：

- 共 `8` 页
- 每页 `128` 字节
- 每个字节控制竖直方向 `8` 个像素

也就是说：

- 一整页就是 `128 x 8`

这就是为什么很多最小工程先按页写数据，而不是直接从“单像素”开始。

### 7.3 为什么英文 8x8 字符容易做

因为：

- 一个 `8x8` 字符正好对应一个页高度
- 你可以按列写 `8` 个 bit
- 非常适合做最小验证和菜单标题

## 9. 为什么第一次要先做最小显示

OLED 类模块的问题通常不是都出在代码逻辑本身，而是容易混在一起：

- 供电不对
- 地址不对
- 接线反了
- 时序不对
- 初始化少了几条命令
- 推挽/开漏方式错了

所以第一次最好的策略永远不是一上来做复杂 UI，而是：

1. 先让屏亮起来
2. 先让屏能稳定显示固定内容
3. 再加交互和页面逻辑

说明：

- 上面这一节说的是独立 OLED 最小验证工程，不是当前 `clock_amd` 主线工程
- 当前 `clock_amd` 主线已经升级为三模式布局、编辑边框、滑动动画，并在 `COUNT` 模式下额外显示 `RUN / STOP`

## 10. 常见错误

### 10.1 `VCC` 接 5V

错误后果：

- 总线被上拉到 5V
- 直接威胁 FPGA IO

正确做法：

- `VCC` 只接 `3.3V`

### 10.2 `SCL/SDA` 接反

错误表现：

- `LED1` 报错
- 屏完全没响应

### 10.3 把 I2C 当推挽 GPIO 用

错误表现：

- 时序不稳定
- 偶尔能跑，偶尔不跑
- 和不同模块搭配时兼容性差

正确做法：

- 开漏
- 输出 `0` 或高阻 `Z`

### 10.4 地址写法混乱

错误表现：

- 代码里写 `0x3C`
- 实际总线上却又按“已经左移后的 0x78”再处理了一次

结果：

- 发错地址

正确做法：

- 先统一你工程里到底使用 `7 位地址` 还是 `8 位写地址`

### 10.5 期望颜色可以改

错误理解：

- 以为程序能把黄蓝双色屏变成全白或全蓝

正确理解：

- 颜色分区是物理属性
- 程序只能控制亮灭和图案

## 11. 排错顺序

如果 OLED 完全不显示，按下面顺序查：

1. `VCC` 是否确实接了 `3.3V`
2. `GND` 是否共地
3. `SCL/SDA` 是否接反
4. 地址是否对
5. I2C 是否按开漏方式实现
6. 初始化命令是否完整
7. 下载的 bitstream 是否是最新

如果 `LED0` 亮、`LED1` 不亮，但显示位置不对或偏移奇怪，优先检查：

1. 页地址设置
2. 列地址起始值
3. 你对字模列顺序和位顺序的理解是否一致

## 12. 最适合它做什么

这块模块最适合做：

- 模式显示
- 参数显示
- 状态页
- 菜单界面
- 计数器/定时器界面
- 简单图标和波形可视化

它不适合做：

- 高帧率图像动画
- 大量复杂汉字滚动但无缓存优化的场景
