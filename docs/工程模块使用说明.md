# CLOCK 工程模块使用说明

本文档面向当前 `clock_amd` 工程，说明当前 Vivado / Nexys A7 100T 版本的：

- 模块结构
- 板级按键与 OLED/蜂鸣器接线
- 模式切换与编辑规则
- 数码管与 OLED 显示规则
- 上板测试步骤

本文档内容已经按“按键统一 UI + OLED 模式显示”的当前版本更新，不再使用旧版 `SW + BTNC` 交互说明。

## 1. 当前工程范围

当前主线工程位于 `clock_amd/`，已经完成：

1. 从旧 Quartus / TEC-8 工程向 Vivado / Nexys A7 100T 的迁移
2. 基于 `100MHz` 单时钟域 + `tick_1k` 使能的时序适配
3. 八位数码管动态扫描显示适配
4. 统一按键 UI 控制层接入
5. OLED 当前模式显示接入
6. 闹铃蜂鸣器接入

当前版本已经可用的功能：

1. 24 小时制正常走时
2. 校时功能，支持小时和分钟加减
3. 闹铃设置，支持小时、分钟和使能开关编辑
4. 闹铃到点 `DP` 提醒和蜂鸣器报警
5. 倒计时设置，支持 `HH:MM:SS` 的时分秒编辑
6. 倒计时启动、停止、到零自动停止
7. OLED 以“三模式布局”显示当前模式、相邻模式和编辑边框
8. 编辑时当前字段闪烁提示

当前仍为占位态、暂不计入正式功能通过项：

1. 12/24 小时制模式
2. 作息提醒模式

## 2. 当前模块结构

### 2.1 板级适配与显示模块

| 文件 | 模块名 | 作用 |
| --- | --- | --- |
| `clock_amd_top.v` | `clock_amd_top` | Nexys A7 顶层，完成板级时钟、按键、OLED、蜂鸣器和数码管接口映射 |
| `nexys_seg_scan.v` | `nexys_seg_scan` | 将内部 6 位显示数据组织为 Nexys A7 八位低有效动态扫描输出 |
| `oled_ui_display.v` | `oled_ui_display` | 通过 SSD1306 I2C OLED 显示三模式布局、编辑边框和左右滑动过渡动画 |
| `i2c_master_simple.v` | `i2c_master_simple` | OLED 的底层 I2C 开漏驱动 |

### 2.2 功能主线模块

| 文件 | 模块名 | 作用 |
| --- | --- | --- |
| `clock.v` | `clock` | 主线集成模块，连接 UI、时间、闹铃、倒计时和显示模块 |
| `ui_ctrl.v` | `ui_ctrl` | 统一按键 UI 控制层，负责模式切换、进入/退出编辑、字段切换、增减脉冲和闪烁节拍 |
| `button_pulse.v` | `button_pulse` | 单个按键的同步、消抖和单次脉冲输出 |
| `clk_ring.v` | `clk_ring` | 在 `tick_1k` 基础上生成 `1Hz` 计时脉冲 |
| `time_core.v` | `time_core` | 保存当前时间，负责正常走时和校时加减 |
| `alarm_ctrl.v` | `alarm_ctrl` | 保存闹铃时分秒和使能位，并完成当前时间与设定闹铃的实时比较和蜂鸣器报警序列 |
| `countdown_ctrl.v` | `countdown_ctrl` | 保存倒计时 `HH:MM:SS`，负责设置、启动、停止和到零停止 |
| `display_ctrl.v` | `display_ctrl` | 统一组织六位数码管显示内容，并在编辑时对选中字段做闪烁隐藏 |
| `seg_7.v` | `seg_7` | 4 位 BCD 到七段码译码 |

### 2.3 当前时序组织方式

旧平台直接提供 `1kHz` 时钟输入，当前平台使用板载 `100MHz`。

当前版本采用：

1. `CLK100MHZ` 作为唯一主时钟
2. 在 `clock_amd_top.v` 中生成 `tick_1k`
3. 各功能模块统一运行在 `CLK100MHZ` 时钟域内，仅在 `tick_1k` 上更新状态
4. `clk_ring.v` 再基于 `tick_1k` 生成 `1Hz` 走时脉冲

这样可以避免在 Vivado 中使用普通寄存器分频结果直接充当新时钟，降低派生慢时钟导致的时序风险。

## 3. 当前板上接口映射

### 3.1 按键

| 板上信号 | FPGA 引脚 | 当前用途 |
| --- | --- | --- |
| `CPU_RESETN` | `C12` | 系统复位，低有效 |
| `BTNL` | `P17` | 左切换：非编辑时切上一模式，编辑时切上一字段 |
| `BTNR` | `M17` | 右切换：非编辑时切下一模式，编辑时切下一字段 |
| `BTNU` | `M18` | 数值加一 / 启动 |
| `BTND` | `P18` | 数值减一 / 停止 |
| `BTNC` | `N17` | 进入编辑 / 退出编辑 |

### 3.2 OLED 接线

当前 OLED 为 `0.96 寸 128x64 SSD1306 IIC` 模块，推荐接线如下：

| OLED 模块引脚 | 开发板连接 | 说明 |
| --- | --- | --- |
| `GND` | `Nexys A7 GND` | 共地 |
| `VCC` | `Nexys A7 3.3V` | 必须接 `3.3V` |
| `SCL` | `JB1 / D14` | I2C 时钟 |
| `SDA` | `JB2 / F16` | I2C 数据 |

说明：

1. 当前 OLED 采用 I2C 开漏方式驱动
2. 不要把模块 `VCC` 接到 `5V`
3. `SCL/SDA` 不要接反

### 3.3 外置蜂鸣器接线

当前接入的是三引脚有源蜂鸣器模块，丝印通常为：

- `GND`
- `I/O`
- `VCC`

推荐接线如下：

| 蜂鸣器模块引脚 | 开发板连接 |
| --- | --- |
| `GND` | `Nexys A7 GND` |
| `VCC` | `Nexys A7 3.3V` |
| `I/O` | `JA1 / C17` |

接线说明：

1. 当前蜂鸣器模块按低电平触发
2. 顶层代码已经做了极性翻转，内部报警信号有效时会主动把 `I/O` 拉低
3. 必须和开发板共地
4. `I/O` 和 `VCC` 不要接反

## 4. 模式顺序与统一 UI 规则

### 4.1 模式顺序

当前模式顺序为：

`正常时钟 -> 校时 -> 闹铃 -> 12/24 小时制占位 -> 倒计时 -> 作息提醒占位 -> 正常时钟`

非编辑状态下：

1. `BTNL`：切到上一模式
2. `BTNR`：切到下一模式
3. `BTNC`：若当前模式可编辑，则进入编辑态

### 4.2 编辑态通用规则

进入编辑态后：

1. `BTNL`：切到上一字段
2. `BTNR`：切到下一字段
3. `BTNU`：当前字段加一
4. `BTND`：当前字段减一
5. `BTNC`：退出编辑态

当前只有以下模式支持进入编辑：

1. 校时模式
2. 闹铃模式
3. 倒计时模式

### 4.3 编辑闪烁规则

进入编辑态后，当前选中的字段会以约 `2Hz` 频率闪烁。

当前闪烁方式为：

1. 校时模式：闪烁当前选中的“小时”或“分钟”
2. 闹铃模式：闪烁当前选中的“小时”或“分钟”或“秒”或“使能位”
3. 倒计时模式：闪烁当前选中的“小时”或“分钟”或“秒”

## 5. 数码管和 OLED 显示说明

### 5.1 数码管位序

当前工程使用 6 位显示，另外 2 位留空。

| 数码管位 | 当前内容 |
| --- | --- |
| `AN0` | 秒个位 |
| `AN1` | 秒十位 |
| `AN2` | 分个位 |
| `AN3` | 分十位 |
| `AN4` | 时个位 |
| `AN5` | 时十位 |
| `AN6` | 空白 |
| `AN7` | 空白 |

说明：

1. Nexys A7 数码管为共阳极动态扫描
2. 位选和段选都为低有效
3. 当前工程已在 `nexys_seg_scan.v` 中完成极性翻转和动态扫描
4. `DP` 当前复用为闹铃提醒输出

### 5.2 OLED 显示规则

OLED 当前用于显示模式层信息，布局已经不是旧版的单行模式名，而是以当前模式为中心的三模式视图。

当前规则如下：

1. 中间显示当前模式，使用放大字形
2. 左侧显示上一模式，使用较小字形
3. 右侧显示下一模式，使用较小字形
4. 进入编辑态后，会在中间当前模式外围绘制矩形边框
5. 按 `BTNL/BTNR` 切换模式时，OLED 会做左右滑动过渡
6. 当前动画已经按整帧刷新边界更新，避免模式字样出现上下错位撕裂
7. 当当前模式为 `COUNT` 时，OLED 底部会额外显示 `RUN` 或 `STOP`

模式切换方向规则：

1. 按 `BTNR` 时，当前模式向左滑出，目标模式从右侧滑入
2. 按 `BTNL` 时，当前模式向右滑出，目标模式从左侧滑入

当前模式名对应如下：

| `mode_state` | OLED 显示 |
| --- | --- |
| `3'b000` | `CLOCK` |
| `3'b001` | `TIME` |
| `3'b010` | `ALARM` |
| `3'b011` | `HOUR` |
| `3'b100` | `COUNT` |
| `3'b101` | `SCHED` |

### 5.3 各模式下的数码管显示

| 当前模式 | 数码管显示 | 说明 |
| --- | --- | --- |
| 正常时钟 | `HH:MM:SS` | 正常走时显示 |
| 校时 | `HH:MM:SS` | 进入编辑后当前字段闪烁 |
| 闹铃 | `HH:MM:SS` | 进入编辑后当前字段闪烁；`DP` 在闹铃模式下用于显示使能状态 |
| 倒计时 | `HH:MM:SS` | 进入编辑后当前字段闪烁 |
| 12/24 占位 | 当前时间 | 仅占位 |
| 作息提醒占位 | 当前时间 | 仅占位 |

## 6. 当前功能与交互规则

### 6.1 正常时钟模式

行为：

1. 复位后回到 `00:00:00`
2. 系统按 24 小时制正常走时
3. `23:59:59 -> 00:00:00`
4. OLED 中间显示 `CLOCK`，左右显示相邻模式

操作：

1. `BTNL/BTNR` 切换模式
2. 当前模式下 `BTNC` 不进入编辑

### 6.2 校时模式

OLED：

1. 非编辑时中间显示 `TIME`
2. 编辑时 `TIME` 外围出现矩形边框
3. 左右继续显示相邻模式小字标题

字段顺序：

1. 小时
2. 分钟
3. 秒

操作：

1. 非编辑时按 `BTNC` 进入编辑
2. 编辑时 `BTNL/BTNR` 在“小时/分钟/秒”之间切换
3. 编辑时 `BTNU` 使当前字段加一
4. 编辑时 `BTND` 使当前字段减一
5. 小时范围：`00~23`
6. 分钟范围：`00~59`
7. 秒范围：`00~59`
8. 编辑时当前字段闪烁
9. 按 `BTNC` 退出编辑

### 6.3 闹铃模式

OLED：

1. 非编辑时中间显示 `ALARM`
2. 编辑时 `ALARM` 外围出现矩形边框

数码管显示：

1. 时十、时个：闹铃小时
2. 分十、分个：闹铃分钟
3. 秒十、秒个：闹铃秒
4. `DP`：闹铃模式下表示使能状态，亮为使能、灭为关闭；编辑使能字段时会跟随闪烁

字段顺序：

1. 小时
2. 分钟
3. 秒
4. 使能位

操作：

1. 非编辑时按 `BTNC` 进入编辑
2. 编辑时 `BTNL/BTNR` 切换字段
3. 编辑时 `BTNU` 使当前字段加一或置 `1`
4. 编辑时 `BTND` 使当前字段减一或置 `0`
5. 小时范围：`00~23`
6. 分钟范围：`00~59`
7. 秒范围：`00~59`
8. 使能位范围：`0/1`
9. 编辑时当前字段闪烁

提醒规则：

1. 当闹铃开启且当前 `HH:MM:SS` 等于设定闹铃 `HH:MM:SS` 时，会触发到点事件
2. 同一次到点事件会触发一次蜂鸣器报警序列
3. 当前蜂鸣器报警序列为：`嘀嘀嘀 停 嘀嘀嘀 停 嘀嘀嘀`
4. 非闹铃模式下，`DP` 继续用于输出 `alarm_match` 提醒
5. 闹铃模式下，`DP` 当前优先用于显示闹铃使能状态

### 6.4 倒计时模式

OLED：

1. 非编辑时中间显示 `COUNT`
2. 编辑时 `COUNT` 外围出现矩形边框
3. OLED 底部会显示 `RUN` 或 `STOP`

数码管显示：

1. 时十、时个显示倒计时小时
2. 分十、分个显示倒计时分钟
3. 秒十、秒个显示倒计时秒

字段顺序：

1. 小时
2. 分钟
3. 秒

操作：

1. 非编辑时按 `BTNC` 进入编辑
2. 编辑时 `BTNL/BTNR` 切换字段
3. 编辑时 `BTNU`：当前字段加一
4. 编辑时 `BTND`：当前字段减一
5. 小时范围：`00~23`
6. 分钟范围：`00~59`
7. 秒范围：`00~59`
8. 非编辑态下：
   `BTNU`：若当前值不为 `00:00:00`，启动或继续倒计时
   `BTND`：停止倒计时
9. 倒计时运行到 `00:00:00` 时自动停止

### 6.5 占位模式

当前仍保留两个占位模式：

1. `HOUR`：12/24 小时制模式占位
2. `SCHED`：作息提醒模式占位

当前测试要求仅为：

1. 模式切入正常
2. OLED 模式名正确
3. 数码管显示不异常
4. 系统不死机

## 7. 上板测试流程

建议按以下顺序测试：

1. 基础显示与复位
2. OLED 模式显示与滑动动画
3. 模式切换
4. 校时模式
5. 闹铃模式
6. 倒计时模式
7. 占位模式检查
8. 蜂鸣器专项检查

## 8. 详细测试步骤

### 8.1 基础显示与复位测试

步骤：

1. 下载 bitstream
2. 按一次 `CPU_RESETN`
3. 观察数码管
4. 持续观察几秒

预期结果：

1. 数码管回到 `00:00:00`
2. 秒正常递增
3. 无全灭、全亮、严重鬼影或无规律闪烁

记录项：

- [ ] 复位后显示 `00:00:00`
- [ ] 秒正常递增
- [ ] 位序正确
- [ ] 段码正确
- [ ] 无明显闪烁/鬼影异常

### 8.2 OLED 与模式切换测试

步骤：

1. 上电后观察 OLED
2. 按 `BTNR` 连续切模式
3. 再按 `BTNL` 反向切回
4. 分别停在 `CLOCK/TIME/ALARM/HOUR/COUNT/SCHED`
5. 观察中间模式是否为大字，左右是否为相邻模式小字
6. 连续多次左右切换，观察滑动过程中是否存在上下撕裂或明显卡顿

预期结果：

1. OLED 正常点亮
2. 中间模式名和当前模式一致
3. 左右显示的是当前模式的相邻模式
4. 左右切换顺序正确
5. 切换时存在明确的左右滑动过渡
6. 当前版本不应再出现单词上下半截明显错位
7. 在 `COUNT` 模式下，OLED 底部能正确显示 `RUN/STOP`

记录项：

- [ ] OLED 正常点亮
- [ ] `CLOCK` 显示正确
- [ ] `TIME` 显示正确
- [ ] `ALARM` 显示正确
- [ ] `HOUR` 显示正确
- [ ] `COUNT` 显示正确
- [ ] `SCHED` 显示正确
- [ ] 中间大字显示正确
- [ ] 左右相邻模式显示正确
- [ ] `BTNL/BTNR` 切换顺序正确
- [ ] 左滑动画正常
- [ ] 右滑动画正常
- [ ] 无明显上下撕裂
- [ ] `COUNT` 模式 `RUN/STOP` 状态显示正确

### 8.3 校时模式测试

步骤：

1. 切到 `TIME`
2. 按 `BTNC` 进入编辑
3. 观察 OLED 是否为 `TIME` 加边框状态
4. 观察小时字段是否闪烁
5. 按 `BTNU`，观察小时加一
6. 按 `BTND`，观察小时减一
7. 按 `BTNR` 切到分钟字段
8. 观察分钟字段是否闪烁
9. 按 `BTNU/BTND` 测试分钟加减
10. 按 `BTNR` 切到秒字段
11. 观察秒字段是否闪烁
12. 按 `BTNU/BTND` 测试秒加减
13. 按 `BTNC` 退出编辑

记录项：

- [ ] 可进入编辑
- [ ] OLED 编辑边框显示正确
- [ ] 小时字段闪烁正确
- [ ] 分钟字段闪烁正确
- [ ] 小时加法正确
- [ ] 小时减法正确
- [ ] 分钟加法正确
- [ ] 分钟减法正确
- [ ] 秒加法正确
- [ ] 秒减法正确
- [ ] 退出编辑正常

### 8.4 闹铃模式测试

步骤：

1. 切到 `ALARM`
2. 按 `BTNC` 进入编辑
3. 依次测试小时、分钟、秒、使能位字段切换
4. 用 `BTNU/BTND` 分别测试对应字段修改

到点提醒测试：

1. 记下当前正常时间
2. 在闹铃模式将闹铃设为“当前时间若干秒后”
3. 使能闹铃
4. 回到正常模式等待
5. 观察到点时蜂鸣器是否按 `嘀嘀嘀 停 嘀嘀嘀 停 嘀嘀嘀` 发声
6. 观察非闹铃模式下 `DP` 是否给出匹配提示

记录项：

- [ ] 可进入编辑
- [ ] 小时字段可设置
- [ ] 分钟字段可设置
- [ ] 秒字段可设置
- [ ] 使能位可设置
- [ ] 字段闪烁正确
- [ ] 闹铃模式下 `DP` 使能显示正确
- [ ] 非闹铃模式下到点后 `DP` 提示正确
- [ ] 到点后蜂鸣器报警节奏正确

### 8.5 倒计时模式测试

建议先从短时间开始，例如 `00:00:05` 或 `00:00:10`。

步骤：

1. 切到 `COUNT`
2. 按 `BTNC` 进入编辑
3. 依次测试小时、分钟、秒三个字段的切换与加减
4. 按 `BTNC` 退出编辑
5. 观察 OLED 底部是否显示 `STOP`
6. 按 `BTNU` 启动
7. 观察 OLED 底部是否切成 `RUN`
8. 观察数码管是否开始递减
9. 按 `BTND` 停止
