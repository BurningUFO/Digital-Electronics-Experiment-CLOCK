# CLOCK 工程模块使用说明

本文档面向当前 `clock_amd` 工程，说明 Vivado / Nexys A7 100T 版本的模块结构、板级接口、统一交互、显示规则和后续上板测试清单。

当前文档按 `SW0` 浏览 / 设置层、八位数码管统一显示、OLED 状态副屏、统一提醒系统和 ADT7420 温度读取链路更新。旧版 `BTNC` 进入 / 退出编辑、OLED 三模式滑动动画、`HOUR / SCHED` 占位等说明已废弃。

## 1. 当前工程范围

当前主线工程位于 `clock_amd/`，已经完成：

1. 从旧 Quartus / TEC-8 工程向 Vivado / Nexys A7 100T 的迁移。
2. 基于 `100MHz` 单时钟域和 `tick_1k` 使能的时序适配。
3. 八位数码管动态扫描显示适配。
4. `CLOCK / TIME / ALARM / HOUR / COUNT / SCHED / COMM` 七模式统一 UI。
5. `SW0=0` 浏览层、`SW0=1` 设置层。
6. `CLOCK` 日期 / 星期设置、`HOUR` 12/24 小时显示格式。
7. `ALARM` 8 槽位、LED 指示、pending 事件和贪睡。
8. `SCHED` 固定 8 个计划点、槽位开关、最近计划和计划提醒。
9. `notification_ctrl` 统一仲裁倒计时、闹钟和计划提醒，并唯一驱动蜂鸣器。
10. 整点报时短蜂鸣，正常走时进入 `HH:00:00` 时响两段 100 ms 短声，中间间隔 100 ms；闹钟、倒计时和日程提醒优先。
11. OLED 状态副屏基础版，显示日期、温度、最近计划、最近闹钟、倒计时状态和提醒弹窗。
12. OLED 字库覆盖完整可打印 ASCII `0x20..0x7E`，COMM 消息小写字母和常见标点可独立显示。
13. ADT7420 温度读取模块和 `TMP_SCL / TMP_SDA` 顶层引脚已接入。
14. ClockLink Studio USB-UART 通信扩展首版已接入，支持 COMM 模式、消息缓存、预设回复、时间同步，以及闹钟/日程/倒计时 PC 直接控制。

当前 UART 协议、PC 编码器和 FPGA `message_store` 仍以 8-bit 可打印 ASCII 为边界。Unicode/中文需要 UTF-8 或码点解析、扩展消息存储和外部/大容量字模 ROM，不属于当前可综合首版范围。

当前已通过全源语法、通信 XSim 回归、PC pytest 和 Vivado 综合时序检查；尚未生成 bitstream，尚未进行 Nexys A7 100T 板级实测。

### 2.3 当前时序组织方式

当前版本采用：

1. `CLK100MHZ` 作为唯一主时钟。
2. 在 `clock_amd_top.v` 中生成 `tick_1k`。
3. 各功能模块统一运行在 `CLK100MHZ` 时钟域内，仅在需要时使用 `tick_1k` 作为更新使能。
4. `clk_ring.v` 基于 `tick_1k` 生成 `1Hz` 走时脉冲。
5. ALARM 和 SCHED 最近事件选择已改为分时扫描并寄存输出，避免长组合路径。
6. OLED 中心模式页已降复杂度，不再保留旧滑动动画，以满足 100MHz 综合时序。

## 3. 当前板上接口映射

### 3.1 按键和开关

| 板上信号 | FPGA 引脚 | 当前用途 |
| --- | --- | --- |
| `CPU_RESETN` | `C12` | 系统复位，低有效 |
| `SW0` | `J15` | 非 COMM 模式下 `0` 为浏览层、`1` 为设置层；COMM 模式下作为最新消息选择 |
| `BTNL` | `P17` | 浏览层切上一模式；设置层切上一字段或槽位；闹钟提醒下用于贪睡 |
| `BTNR` | `M17` | 浏览层切下一模式；设置层切下一字段或槽位；闹钟提醒下用于贪睡 |
| `BTNU` | `M18` | 设置层数值加一；COUNT 浏览层启动；闹钟提醒下用于贪睡 |
| `BTND` | `P18` | 设置层数值减一；COUNT 浏览层停止；闹钟提醒下用于贪睡 |
| `BTNC` | `N17` | 上下文确认；ALARM/SCHED 开关切换；提醒激活时确认 / 消音 |

### 3.2 数码管、LED 和蜂鸣器

| 板上信号 | FPGA 引脚 / 端口 | 当前用途 |
| --- | --- | --- |
| `AN[7:0]` | 见 `.xdc` | 八位数码管位选，低有效 |
| `CA..CG, DP` | 见 `.xdc` | 七段码和小数点输出，低有效 |
| `LED0~LED7` | 见 `.xdc` | ALARM/SCHED 槽位开关和选中状态提示 |
| `LED8~LED15` | 见 `.xdc` | 当前未使用，保持 0 |
| `BUZZER_IO` | `C17 / JA1` | 外置低电平触发有源蜂鸣器 |

整点报时规则：

1. 只在正常自动走时的小时边界触发，即从 `MM:SS=59:59` 进入下一小时 `00:00`。
2. 蜂鸣器输出两段短声，每段约 100 ms，中间间隔约 100 ms。
3. 整点报时不设置 `notify_active`，不锁定按键，不显示 OLED 提醒弹窗。
4. 若同一时刻存在倒计时、闹钟或日程提醒，统一提醒优先，整点报时被覆盖。

蜂鸣器推荐接线：

| 蜂鸣器模块引脚 | 开发板连接 |
| --- | --- |
| `GND` | `Nexys A7 GND` |
| `VCC` | `Nexys A7 3.3V` |
| `I/O` | `JA1 / C17` |

### 3.3 OLED 和温度传感器

外接 OLED 为 `0.96 寸 128x64 SSD1306 IIC` 模块，推荐接线如下：

| OLED 模块引脚 | 开发板连接 | 说明 |
| --- | --- | --- |
| `GND` | `Nexys A7 GND` | 共地 |
| `VCC` | `Nexys A7 3.3V` | 不要接 `5V` |
| `SCL` | `JB1 / D14` | 外接 OLED I2C 时钟 |
| `SDA` | `JB2 / F16` | 外接 OLED I2C 数据 |

ADT7420 使用 Nexys A7 板载温度传感器接口：

| 信号 | FPGA 引脚 | 说明 |
| --- | --- | --- |
| `TMP_SCL` | `C14` | ADT7420 独立 I2C 时钟 |
| `TMP_SDA` | `C15` | ADT7420 独立 I2C 数据 |

外接 OLED 的 `D14/F16` 和板载 ADT7420 的 `C14/C15` 不共用同一组 I2C 线。

### 3.4 USB-UART

Nexys A7 J6 USB-UART 用于 ClockLink Studio PC 通信：

| 信号 | FPGA 引脚 | 说明 |
| --- | --- | --- |
| `UART_RXD` | `C4` | PC 到 FPGA，115200 8N1 |
| `UART_TXD` | `D4` | FPGA 到 PC，115200 8N1 |

协议详见 `docs/UART_PROTOCOL.md`。

## 4. 模式顺序与统一 UI 规则

当前模式顺序为：

`CLOCK -> TIME -> ALARM -> HOUR -> COUNT -> SCHED -> COMM -> CLOCK`

浏览层，`SW0=0`：

1. `BTNL / BTNR` 切换模式。
2. `COUNT` 模式下，`BTNU` 启动或继续倒计时，`BTND` 停止倒计时。
3. `COMM` 模式下，`SW0-SW15` 低位优先选择最近 16 条消息，`BTNU/BTND` 滚动正文，`BTNC` 切换查看/回复，`BTNR` 发送预设回复。
4. 其他模式下，`BTNU / BTND` 在浏览层不修改核心数据。

设置层，`SW0=1`：

1. `BTNL / BTNR` 切换当前模式的字段或槽位。
2. `BTNU / BTND` 修改当前字段、槽位或开关值。
3. `BTNC` 用于当前字段的上下文确认或开关切换。
4. 当前选中字段按约 `2Hz` 闪烁。
5. `COMM` 模式不使用 `SW0` 设置层；所有开关都解释为消息槽选择。

提醒激活时：

1. 普通模式切换和设置层操作被锁定。
2. `BTNC` 优先作为提醒确认 / 消音。
3. 闹钟提醒下，方向键用于提交贪睡请求。

## 5. 数码管和 OLED 显示说明

### 5.1 八位数码管结构

当前工程使用 8 位字符码总线，显示结构为：

`模式 / 状态 / HH / MM / SS`

| 数码管位 | 当前内容 |
| --- | --- |
| `AN0` | 秒个位 |
| `AN1` | 秒十位 |
| `AN2` | 分个位 |
| `AN3` | 分十位 |
| `AN4` | 时个位或当前模式数据个位 |
| `AN5` | 时十位或当前模式数据十位 |
| `AN6` | 状态字符 |
| `AN7` | 模式字符 |

模式字符：

| 模式 | `AN7` |
| --- | --- |
| `CLOCK` | `n` |
| `TIME` | `t` |
| `ALARM` | `A` |
| `HOUR` | `H` |
| `COUNT` | `C` |
| `SCHED` | `S` |
| `COMM` | 左四位 `COMM` |

状态字符按模式变化，例如 `HOUR` 显示 `1/2` 表示 12/24 小时制，`COUNT` 显示 `r/P` 表示运行 / 停止，`ALARM/SCHED` 显示 `o/F` 表示最近或当前槽位开关状态。
COMM 模式下八位数码管左四位显示 `COMM`，右四位显示 `DISC / WAIT / CONN / MSG! / ERR `。

### 5.2 OLED 显示规则

OLED 当前是状态副屏基础版，不再显示旧版三模式滑动动画、左右相邻模式和编辑边框。

普通页面显示：

1. 日期 / 星期。
2. 温度；读取成功显示整数摄氏度，读取失败显示 `--C`。
3. 最近一个已开启计划点。
4. 最近一个已开启闹钟。
5. 倒计时 `RUN / STOP` 状态。
6. 当前模式的普通 8x 文本标识。
7. COMM 模式下改为通信专用页面，显示 ClockLink 状态、消息时间戳/正文窗口或预设回复列表。

OLED 字库范围：

1. 内置 8 列、7 行点阵字形覆盖可打印 ASCII `0x20..0x7E`。
2. 小写 `a-z` 使用独立小写字形，不再直接映射到大写。
3. Unicode/中文不在当前 FPGA 字库与 UART 协议范围内；PC 发送消息仍会拒绝非 ASCII 正文。

提醒激活时，OLED 覆盖普通页面显示提醒弹窗：

1. `COUNT DONE`
2. `ALARM N`
3. `SCHED N`

其中 `N` 表示对应槽位或计划编号。

### 5.3 各模式下的数码管显示

| 当前模式 | 浏览层显示 | 设置层显示 |
| --- | --- | --- |
| `CLOCK` | 当前时间，按 HOUR 设置使用 12/24 显示 | 月、日、星期 |
| `TIME` | 当前时间，按 HOUR 设置使用 12/24 显示 | 原始 24 小时制时、分、秒 |
| `ALARM` | 最近一个已开启闹钟；无开启时显示 `00:00:00` | 当前槽位时、分、秒和开关状态 |
| `HOUR` | 当前显示格式，`1` 表示 12 小时，`2` 表示 24 小时 | 同浏览层，可切换格式 |
| `COUNT` | 倒计时剩余值和运行 / 停止状态 | 倒计时时、分、秒 |
| `SCHED` | 最近一个已开启计划点；无开启时显示 `00:00:00` | 当前计划槽位时间和开关状态 |
| `COMM` | 左四位 `COMM`，右四位通信状态 | 不进入普通设置层 |

`TIME` 和 `ALARM` 设置层始终使用 24 小时制编辑；12/24 小时制只影响显示，不影响内部计时、事件比较或倒计时。

## 6. 各模式交互规则

### 6.1 CLOCK

浏览层：

1. 显示当前时间。
2. `BTNL / BTNR` 切换模式。

设置层，`SW0=1`：

1. 字段顺序：月 -> 日 -> 星期。
2. `BTNL / BTNR` 切换字段。
3. `BTNU / BTND` 修改字段。
4. 日期和星期数据供 OLED 使用。

### 6.2 TIME

设置层，`SW0=1`：

1. 字段顺序：小时 -> 分钟 -> 秒。
2. `BTNL / BTNR` 切换字段。
3. `BTNU / BTND` 修改字段。
4. 设置层显示和编辑原始 24 小时制。

### 6.3 ALARM

设置层，`SW0=1`：

1. 字段顺序：槽位 -> 小时 -> 分钟 -> 秒 -> 开关。
2. `BTNL / BTNR` 切换字段。
3. 在槽位字段，`BTNU / BTND` 切换 8 个闹钟槽位。
4. 在时间字段，`BTNU / BTND` 修改时、分、秒。
5. 在开关字段，`BTNU / BTND` 或 `BTNC` 切换当前槽位开启 / 关闭。
6. LED0~LED7 表示槽位开关和当前选中状态。

提醒规则：

1. 开启槽位到点后产生 pending 事件。
2. 多个同秒闹钟按槽位编号顺序处理。
3. `notification_ctrl` 接管蜂鸣器、OLED 弹窗、确认和贪睡。
4. 闹钟提醒激活时，`BTNC` 关闭当前提醒，方向键提交贪睡请求。

### 6.4 HOUR

`HOUR` 模式用于切换全局显示格式：

1. `12/24` 只影响显示。
2. 内部时间、闹钟比较、计划比较和倒计时均保持 24 小时制。
3. `TIME` 和 `ALARM` 设置层仍使用 24 小时制编辑。
4. 设置层下 `BTNU / BTND / BTNC` 均可切换 12/24 小时制。

### 6.5 COUNT

浏览层：

1. `BTNU` 启动或继续倒计时。
2. `BTND` 停止倒计时。

设置层，`SW0=1`：

1. 字段顺序：小时 -> 分钟 -> 秒。
2. `BTNL / BTNR` 切换字段。
3. `BTNU / BTND` 修改字段。
4. 倒计时运行到 `00:00:00` 时自动停止并进入统一提醒系统。

### 6.6 SCHED

SCHED 首版采用固定 8 个计划点：

| 编号 | 时间 | OLED 名称 |
| --- | --- | --- |
| 0 | `08:00` | `CLASS 1` |
| 1 | `09:40` | `BREAK` |
| 2 | `10:00` | `CLASS 2` |
| 3 | `11:40` | `LUNCH` |
| 4 | `14:00` | `CLASS 3` |
| 5 | `15:40` | `BREAK` |
| 6 | `19:00` | `STUDY` |
| 7 | `21:30` | `REST` |

设置层，`SW0=1`：

1. 字段顺序：槽位 -> 开关。
2. 在槽位字段，`BTNU / BTND` 切换 8 个固定计划点。
3. 在开关字段，`BTNU / BTND` 或 `BTNC` 切换当前计划点开启 / 关闭。
4. LED0~LED7 表示计划点开关和当前选中状态。

计划到点后进入统一提醒系统，OLED 显示计划提醒弹窗，`BTNC` 可提前确认。

### 6.7 COMM

COMM 模式用于 PC 上位机 ClockLink Studio 通信：

1. `SW0-SW15` 查看最近 16 条消息，低位优先；`SW0` 是最新消息。
2. 多个开关同时打开时选择最低位。
3. OLED 显示消息时间戳和正文窗口，正文过长时用 `BTNU/BTND` 滚动。
4. 查看有效消息时按 `BTNC` 进入预设回复选择，再按 `BTNC` 回到查看消息。
5. 回复模式下 `BTNU/BTND` 选择 8 条固定回复，`BTNR` 通过 UART 发送 `REPLY` 帧给 PC。
6. PC 可通过协议直接同步时间，读写闹钟/日程槽，设置、启动和停止倒计时。

## 8. 上板测试清单

建议生成 bitstream 后按以下顺序测试：

1. 复位和基础走时：`CPU_RESETN` 后显示回到初始状态，秒正常递增。
2. 七模式切换：`CLOCK / TIME / ALARM / HOUR / COUNT / SCHED / COMM` 顺序正确。
3. `SW0` 设置层：每个模式进入 / 退出设置层稳定，字段闪烁正确。
4. 八位数码管：模式位、状态位、六位数据位、`DP` 显示正确。
5. `CLOCK` 日期 / 星期设置：OLED 日期同步显示。
6. `TIME` 校时：小时、分钟、秒可设置，退出设置层后继续走时。
7. `HOUR` 格式：12/24 显示切换正确，TIME/ALARM 编辑仍为 24 小时制。
8. `ALARM` 8 槽位：槽位选择、开关、LED 指示、最近闹钟显示、到点提醒和贪睡。
9. `COUNT` 倒计时：编辑、启动、停止、到零提醒。
10. `SCHED` 固定计划：槽位选择、开关、LED 指示、最近计划显示和计划提醒。
11. 统一提醒：倒计时、闹钟、计划三类事件的优先级、蜂鸣节奏、`BTNC` 确认 / 消音。
12. 整点报时：把时间调到任意 `HH:59:55` 附近，确认进入下一小时 `HH+1:00:00` 时输出两段短蜂鸣，且不出现提醒弹窗。
13. OLED 状态副屏：日期、温度、最近计划、最近闹钟、倒计时状态和提醒弹窗。
14. OLED ASCII 字库：通过 COMM 消息发送大小写、数字和标点组合，例如 `Hello, FPGA! @ 12:34?`，确认 OLED 不再空白或全部大写化。
15. ADT7420 温度：温度读数是否合理稳定；若异常，优先检查 `TMP_SCL / TMP_SDA` 和采样时序。
16. COMM 模式：数码管 `COMM + 状态`、OLED 通信页面、PC 时间同步、消息发送、SW 查看、滚动和预设回复。
17. PC 控制：ClockLink Studio 读写闹钟/日程槽，设置、启动、停止倒计时。
