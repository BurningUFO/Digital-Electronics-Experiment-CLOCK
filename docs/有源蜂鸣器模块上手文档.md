## 1. 模块是什么

这不是裸蜂鸣器，而是已经带了驱动电路的成熟模块。

它和“无源蜂鸣器”的根本区别是：

- 这个模块内部已经能自己发声
- 你不需要给它输出几千赫兹的音频方波
- 你只需要给一个触发电平，它就会响

对于这块模块，触发规则是：

- `I/O = 低电平` 时响
- `I/O = 高电平` 时不响

这就是“低电平触发”的实际含义。

## 2. 引脚定义

模块三个引脚分别是：

- `GND`
  - 接地
- `I/O`
  - 控制输入
- `VCC`
  - 模块供电

不要把 `I/O` 理解成普通电源脚，它是逻辑控制脚。

## 3. 为什么它是低电平触发

从模块原理图可以看出：

- 模块板上用了 `S8550`
- `S8550` 是 `PNP` 三极管
- 它被做成了高边开关结构

这类结构的典型行为是：

- 当基极被拉低时，三极管导通
- 蜂鸣器得到供电，开始鸣叫
- 当基极接近 `VCC` 时，三极管截止
- 蜂鸣器断电，不再鸣叫

所以它天然适合“低电平触发”。

## 4. 最重要的电气结论

这个模块可以直接被 `Nexys A7-100T` 控制，但推荐方案只有一个：

- `VCC` 接 `3.3V`
- `GND` 接 `GND`
- `I/O` 接 FPGA 普通输出引脚

不要默认拿 `5V` 供电后直接把 `I/O` 接 FPGA。

原因是：

- FPGA 最多只能稳定输出 `3.3V`
- 这个模块的高边 `PNP` 关断要求控制脚接近 `VCC`
- 如果模块 `VCC=5V`，那么 `I/O=3.3V` 未必能把三极管彻底关死
- 结果可能是蜂鸣器一直半响、误响，或者根本关不干净

所以对 `Nexys A7` 最稳妥的方案就是：

- 让模块和 FPGA 都工作在 `3.3V` 逻辑域

## 5. 物理接线

推荐接线：

- `模块 GND -> 开发板 GND`
- `模块 VCC -> 开发板 3.3V`
- `模块 I/O -> FPGA 一个普通输出引脚`

当前最小测试工程使用的是：

- `I/O -> JA1`
- `JA1 -> C17`

如果你沿用现成测试工程，直接按这个接就行。

工程行为：

- `SW0 = 0`
  - 蜂鸣器不响
  - `LED0` 熄灭
- `SW0 = 1`
  - 蜂鸣器响
  - `LED0` 点亮

这个设计故意做得很简单，方便把问题压缩到：

- 接线是否对
- 供电是否对
- 约束是否对
- 模块本身是否正常

## 7. 为什么代码里要取反

因为模块是低电平触发。

如果你希望：

- 开关打开时响
- 开关闭合表示“启用”

那么代码通常会写成：

```verilog
assign buzzer_io = ~sw0;
```

意思就是：

- `sw0 = 1` 时，把输出变成 `0`
- 让模块被低电平触发

## 8. 这个模块不需要什么

对于这块有源模块，你通常不需要：

- PWM 音频波形发生器
- 分频器去生成 `2kHz` 或 `4kHz`
- 专门的蜂鸣器驱动 IP

这些更常见于“无源蜂鸣器”。

本模块更像一个“会发声的数字输出器件”。

## 9. 模块上贴纸和正极标记是什么意思

### 9.1 `remove seal after washing`

这个贴纸的意思是：

- 清洗后撕掉封贴

它是生产工艺相关的保护贴，不是装饰件。

实际使用中：

- 如果发声孔上还有这张贴纸，应该撕掉
- 不撕会让声音变小、发闷，甚至几乎不出声

### 9.2 圆圈里的 `+`

这表示蜂鸣器本体本身有极性。

但你在使用模块时，真正要接的是：

- `GND`
- `I/O`
- `VCC`

也就是说：

- 你不需要单独去研究蜂鸣器本体正负极怎么接
- 只要按模块丝印接线即可

## 10. 常见错误

### 10.1 把它当无源蜂鸣器

错误表现：

- 一上来写复杂分频/PWM 代码

正确做法：

- 先直接给低电平触发，看会不会叫

### 10.2 `VCC` 接 `5V`，`I/O` 直连 FPGA

错误表现：

- 关不干净
- 一直叫
- 行为异常

正确做法：

- 先全部按 `3.3V` 方案接

### 10.3 `I/O` 和 `VCC` 接反

错误表现：

- 完全不响
- 模块异常发热

正确做法：

- 严格按丝印接

### 10.4 忘记共地

错误表现：

- 控制信号看起来没问题但模块没反应

正确做法：

- `模块 GND` 必须和 `Nexys A7 GND` 连在一起

## 11. 排错顺序

如果蜂鸣器不响，按下面顺序查：

1. 模块发声孔上的封贴是否已经撕掉
2. `VCC` 是否接了 `3.3V`
3. `GND` 是否共地
4. `I/O` 是否接到了正确的 FPGA 引脚
5. 约束文件端口名是否和顶层一致
6. 顶层模块是否设对
7. 下载的 bitstream 是否是最新的

如果模块一直响，优先怀疑：

1. 逻辑极性写反
2. `VCC` 用了 `5V`
3. `I/O` 管脚悬空或约束错误

## 12. 最适合它做什么

这类模块最适合做：

- 简单提示音
- 按键反馈
- 模式切换提示
- 报警提示
- 倒计时结束提醒

它不适合做：

- 音乐播放
- 多音调精细控制
- 波形合成

## 13. 后续扩展建议

在这个模块上继续开发，推荐按这个顺序扩展：

1. 先做开关触发
2. 再做按键触发
3. 再做短鸣、长鸣、连续鸣叫模式
4. 最后再做状态机里的报警逻辑

例如你可以继续扩展成：

- 开机鸣叫一次
- 计时到零鸣叫三次
- 错误状态快速间歇鸣叫

## 14. 一句话总结

这块模块的使用核心不是“生成音频”，而是：

- 把它当作一个 `低电平触发的有源提示器`
- 用 `3.3V` 供电
- 直接由 FPGA 数字逻辑控制即可
