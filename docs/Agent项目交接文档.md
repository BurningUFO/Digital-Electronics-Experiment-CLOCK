## 5. 当前主线的输入逻辑

项目当前采用“`K1~K5` 选模式，`SWA/SWB/SWC` 选功能，`QD` 执行动作”的方式。

注意：当前代码中的编码顺序是：

```verilog
ctrl_sel = {sw_c, sw_b, sw_a};
```

也就是说，代码解释顺序是 **`SWC SWB SWA`**，不是用户直觉里的 **`SWA SWB SWC`**。

这是一个重要坑点，之前用户已经因为这个顺序误解过一次。

### 当前模式映射

- `k5_mode_schedule = 1`
  - `mode_state = 3'b101`
  - 作息提醒模式
- `k4_mode_countdown = 1`
  - `mode_state = 3'b100`
  - 倒计时模式
- `k3_mode_hour_format = 1`
  - `mode_state = 3'b011`
  - 12/24 小时制模式
- `k2_mode_alarm = 1`
  - `mode_state = 3'b010`
  - 闹铃模式
- `k1_mode_time_set = 1`
  - `mode_state = 3'b001`
  - 校时模式
- `K1~K5` 全为 `0`
  - `mode_state = 3'b000`
  - 正常时钟模式

若多个模式开关同时打开，则按高位优先，也就是 `K5 > K4 > K3 > K2 > K1`。

### 当前功能映射

- `ctrl_sel = 000`
  - 当前未分配公共动作
- `ctrl_sel = 001`
  - 通用选择；在倒计时模式下用于“分钟十位加一”
- `ctrl_sel = 010`
  - 通用加一；在倒计时模式下用于“分钟个位加一”
- `ctrl_sel = 011`
  - 确认 / 启停
- `ctrl_sel = 100`
  - 倒计时模式下“秒十位加一”
- `ctrl_sel = 101`
  - 倒计时模式下“秒个位加一”

### 实际板上操作时的注意

如果用户按“ABC=001”去理解，通常会出错。  
当前代码真正要求的是按 **`SWC SWB SWA`** 理解。

例如：

- 想触发 `key_select_pulse`
  - 代码要求 `SWC SWB SWA = 001`
  - 对应物理开关状态是 `SWA=1, SWB=0, SWC=0`
  - 如果用户按 “A B C = 001” 去拨，就会变成 `SWA=0, SWB=0, SWC=1`，这是错误的

模式本身已经不再依赖 `QD` 循环切换，因此板上调试时先看 `K1~K5` 开关位置，再用 `QD` 执行当前模式内动作即可。

本步骤已经完成，当前主线就是按下面思路实现的：

1. 保留当前主线的顶层结构
2. 新建/重写 `countdown_ctrl.v`
3. 让它只维护倒计时内部状态
4. 通过 `mode_ctrl` 读取独立模式开关并决定什么时候进入倒计时模式
5. 通过 `QD + ctrl_sel` 在倒计时模式下完成四个字段的设置和启停
6. 通过 `display_ctrl` 决定倒计时模式下显示什么

也就是说，倒计时应该作为“功能模块”接进来，而不是把整个顶层重写。
