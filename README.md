# Digital Electronics Experiment CLOCK

## 项目概览

本仓库用于维护数字逻辑与数字系统课程设计中的电子钟项目，当前同时保留两套工程：

1. 旧平台主线：基于 TEC-8 实验箱与 Quartus 的 CPLD 工程
2. 新平台迁移版：基于 Nexys A7 100T 与 Vivado 的 FPGA 工程

当前仓库重点已经从“只维护旧实验箱版本”转为“在保留原主线资料的前提下，完成向 Nexys A7 平台的迁移与验证”。

## 当前目录结构

### 根目录旧平台工程

根目录下这些文件仍对应旧 Quartus 主线：

- `clock.v`
- `clk_ring.v`
- `time_core.v`
- `time_set_ctrl.v`
- `alarm_ctrl.v`
- `countdown_ctrl.v`
- `display_ctrl.v`
- `key_ctrl.v`
- `mode_ctrl.v`
- `seg_7.v`
- `clock.qpf`
- `clock.qsf`

它们用于保留原 TEC-8 / MAX7000S 平台上的实现与历史演进过程。

### `clock_amd/`

`clock_amd/` 是当前的新平台迁移工程，面向：

- 开发板：`Nexys A7 100T`
- 工具链：`Vivado`
- 主时钟：`100 MHz`

当前该目录中保留了需要纳入版本管理的核心内容：

- `clock_amd.xpr`
- `clock_amd.srcs/`
- `迁移说明.md`

其中迁移版的关键适配包括：

1. 把旧工程的核心功能模块迁入 Vivado 工程
2. 增加 `clock_amd_top.v` 作为 Nexys A7 顶层
3. 把旧平台的 `QD/SWA/SWB/SWC/K1~K5` 重新映射到 `BTNC/SW0~SW7`
4. 将原先依赖外部 `1kHz` 时钟的逻辑改为 `100MHz + tick_1k` 的单时钟域结构
5. 新增 `nexys_seg_scan.v` 适配 8 位低有效动态扫描数码管

## 当前已实现功能

无论旧平台主线还是新平台迁移版，当前逻辑主线都已具备以下核心能力：

1. 24 小时制正常走时
2. 秒、分、时三级正确进位
3. 低有效复位清零到 `00:00:00`
4. 校时功能
5. 闹铃设置、开关控制与到点 `DP` 提醒
6. 倒计时设置、启动、暂停和到零停止
7. 模式优先级切换

当前仍为占位态、未作为正式完成项的功能：

1. 12/24 小时制模式
2. 作息提醒模式

## 当前板级映射说明

迁移版 `clock_amd/` 当前使用以下映射：

- `CPU_RESETN`：系统复位
- `BTNC`：统一执行键
- `SW0`：`SWA`
- `SW1`：`SWB`
- `SW2`：`SWC`
- `SW3`：校时模式
- `SW4`：闹铃模式
- `SW5`：12/24 小时制模式占位
- `SW6`：倒计时模式
- `SW7`：作息提醒模式占位

模式优先级为：

`SW7 > SW6 > SW5 > SW4 > SW3`

## 维护说明

当前仓库处于“保留旧平台主线 + 推进新平台迁移”的并行阶段，后续建议遵守以下原则：

1. 新开发优先基于 `clock_amd/` 进行，不再回到旧平台接口上扩展新功能
2. 若需要追溯历史实现、资源压缩取舍或旧板交互逻辑，可参考根目录旧 Quartus 工程
3. 每次结构调整、迁移适配或功能增删，都应同步更新 `docs/功能与修改日志.md`
