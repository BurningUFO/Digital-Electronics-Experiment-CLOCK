## 5. 当前主线的输入逻辑

项目采用“`SWA/SWB/SWC` 选功能，`QD` 执行动作”的方式。

注意：当前代码中的编码顺序是：

```verilog
ctrl_sel = {sw_c, sw_b, sw_a};
```

也就是说，代码解释顺序是 **`SWC SWB SWA`**，不是用户直觉里的 **`SWA SWB SWC`**。

这是一个重要坑点，之前用户已经因为这个顺序误解过一次。

### 当前功能映射

- `ctrl_sel = 000`
  - 模式切换
- `ctrl_sel = 001`
  - 通用选择；在倒计时模式下用于“分钟十位加一”
- `ctrl_sel = 010`
  - 通用加一；在倒计时模式下用于“分钟个位加一”
- `ctrl_sel = 011`
  - 确认 / 启停
- `ctrl_sel = 100`
  - 倒计时模式下“秒十位加一”
- `ctrl_sel = 101`
  - 倒计时模式下“秒个位加一”

### 实际板上操作时的注意

如果用户按“ABC=001”去理解，通常会出错。  
当前代码真正要求的是按 **`SWC SWB SWA`** 理解。

例如：

- 想触发 `key_select_pulse`
  - 代码要求 `SWC SWB SWA = 001`
  - 对应物理开关状态是 `SWA=1, SWB=0, SWC=0`
  - 如果用户按 “A B C = 001” 去拨，就会变成 `SWA=0, SWB=0, SWC=1`，这是错误的

如果后续接手 agent 觉得这个映射太反直觉，可以考虑把 `key_ctrl` 改成按 `SWA SWB SWC` 顺序解释。

## 6. 当前显示行为

### 正常模式

- `LG1`：秒个位
- `LG2-LG6`：其余时间位

### 校时模式

- `LG1 = 1`
  - 当前设置“小时”
- `LG1 = 2`
  - 当前设置“分钟”
- 其余位继续显示当前时间

### 其他扩展模式

- 倒计时模式
  - 显示 `00:MM:SS`
  - `LG6`、`LG5` 固定为 `0`
- 其余扩展模式
  - 当前只是预留框架
  - `LG1` 显示模式编号

## 12. 倒计时模块如果继续做，推荐的接法

本步骤已经完成，当前主线就是按下面思路实现的：

1. 保留当前主线的顶层结构
2. 新建/重写 `countdown_ctrl.v`
3. 让它只维护倒计时内部状态
4. 通过 `mode_ctrl` 决定什么时候进入倒计时模式
5. 通过 `QD + ctrl_sel` 在倒计时模式下完成四个字段的设置和启停
6. 通过 `display_ctrl` 决定倒计时模式下显示什么

也就是说，倒计时应该作为“功能模块”接进来，而不是把整个顶层重写。
